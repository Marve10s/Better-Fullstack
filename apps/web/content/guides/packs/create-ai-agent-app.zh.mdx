当你希望应用和代理工作流一起起步时，请使用 AI Agent App pack。它会准备一个 Next.js 项目，并加入面向 AI CLI 的文档和本地代理工具。

<StarterPackCta id="ai-agent-app" />

## 生成 AI 代理 starter

打开 [Stack Builder 中的 AI Agent App pack](/new?preset=ai-cli-agent-workbench&view=command)，或运行：

```bash
bun create better-fullstack@latest my-ai-agent-app \
  --ecosystem typescript \
  --frontend next \
  --backend self \
  --database sqlite \
  --orm drizzle \
  --auth better-auth \
  --api trpc \
  --ai ai-cli \
  --addons biome ruler turborepo mcp skills \
  --ai-docs claude-md agents-md cursorrules
```

## Pack 包含什么

- Next.js 用于产品界面。
- SQLite 和 Drizzle 用于本地优先的持久化。
- Better Auth 用于账户和会话。
- AI CLI 设置、MCP 支持、skills，以及生成的代理文档。
- Biome 和 Ruler，让代理指令和代码质量规则与仓库放在一起。

## 首先要自定义的内容

1. 定义面向用户的代理任务：研究、编码、支持、自动化或分析。
2. 加入代理允许读取和写入的应用数据模型。
3. 在连接外部服务之前，收紧 MCP/tool 指令。
4. 当协作或生产托管需要时，从 SQLite 迁移到托管数据库。

## 什么时候适合这个 pack

选择它来构建 AI 辅助开发工具、内部自动化、代理仪表盘、工作流助手，或那些应用结构和代理指令需要一起演进的实验。

如需技术栈层面的 walkthrough，请阅读 [Next.js AI CLI 代理工作台](/guides/ai/nextjs-ai-cli-agent-workbench/)。
