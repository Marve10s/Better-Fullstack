Usa este stack cuando quieras una app Expo pensada primero para móvil y no necesites generar un backend dentro del mismo scaffold.

```npm
npm create better-fullstack@latest my-mobile-app -- \
  --ecosystem react-native \
  --frontend native-uniwind \
  --mobile-navigation expo-router \
  --mobile-ui uniwind \
  --mobile-storage mmkv \
  --mobile-testing maestro-react-native-testing-library \
  --mobile-push expo-notifications \
  --mobile-ota expo-updates \
  --mobile-deep-linking expo-linking \
  --ai-docs none \
  --no-install
```

## Qué crea

- Una app nativa Expo con Uniwind.
- Expo Router para rutas nativas basadas en archivos.
- MMKV, React Native Testing Library, Maestro, notificaciones push, actualizaciones OTA y deep linking.
- Sin frontend web, backend, base de datos, ORM, auth ni capa de API por defecto.
- Una base mobile-first que puedes conectar a una API más adelante.

## Forma generada

Este comando crea una base de frontend nativo, no una app web fullstack. Mantiene las decisiones de backend fuera del scaffold inicial para que la app móvil pueda apuntar luego a una API existente o a un servicio generado por separado.

Forma representativa:

```txt
my-mobile-app/
  bts.jsonc
  app/
  components/
  package.json
  maestro/
```

Las rutas exactas de Expo pueden variar según la versión del template, pero la propiedad del código es clara: las pantallas y componentes nativos viven en la app, mientras que las responsabilidades de servidor quedan intencionalmente fuera.

## Pantalla de ejemplo

Uniwind permite usar clases de estilo utilitario en componentes React Native:

```tsx
import { Text, View } from "react-native";

export default function HomeScreen() {
  return (
    <View className="flex-1 items-center justify-center bg-white px-6">
      <Text className="text-2xl font-semibold text-slate-950">
        Mobile app
      </Text>
      <Text className="mt-2 text-center text-slate-600">
        Connect an API when the product contract is ready.
      </Text>
    </View>
  );
}
```

Cuándo agregues un backend, mantén los clientes de API detrás de un módulo pequeño. Así podrás cambiar orígenes locales, de staging y de producción sin tocar las pantallas.

## Compatibilidad

React Native es un ecosistema propio en Better Fullstack. Usa `--ecosystem react-native`; no combines flags de frontend web TypeScript con flags de frontend nativo.

Esta guía deja fuera a propósito backend, base de datos, ORM, auth y API. La autenticación móvil y las APIs de servidor necesitan decisiones específicas de plataforma que conviene tomar cuando los requisitos del producto estén claros.

## Cuándo elegirlo

Elige este stack para prototipos móviles, apps companion y bases nativas donde la API puede crearse aparte.

## Agregar una API después

Los siguientes pasos habituales son una API Hono para un backend TypeScript liviano, una app Next.js o TanStack Start si también necesitas superficie web, o un backend dedicado que ya exista fuera de Better Fullstack.

Antes de agregar una API, decide cómo la app nativa manejará:

- URLs base de API por entorno.
- Almacenamiento de sesión de auth en el dispositivo.
- Refresh tokens o renovación de sesión.
- Comportamiento offline y actualizaciones optimistas.
- Perfiles de build nativos para desarrollo, preview y producción.

## Despliegue

Desplegar con Expo es distinto a desplegar una web. Planifica perfiles de build nativos, identificadores de app, credenciales de firma y variables de entorno usadas en tiempo de build. No pongas secretos de servidor dentro del bundle móvil; las apps nativas deben llamar a un backend que mantenga los secretos del lado servidor.

## Solucion de problemas

- Si no aparecen archivos web generados, es esperado: este stack usa frontend nativo y no genera frontend web.
- Si no hay rutas backend, el comando selecciona explícitamente `--backend none`.
- Si los estilos no se aplican, revisa la configuración de Uniwind y el escaneo de clases en la app generada.
- Si las llamadas a la API funcionan en el simulador pero no en el dispositivo, comprueba que la URL base apunte a un host accesible desde el dispositivo.
