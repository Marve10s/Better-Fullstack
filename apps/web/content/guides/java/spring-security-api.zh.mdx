当你需要一个带持久化、安全、数据库迁移和 integration-test tooling 的 Spring Boot backend 时，使用这个 stack。

```npm
npm create better-fullstack@latest my-secure-spring-api -- \
  --ecosystem java \
  --java-web-framework spring-boot \
  --java-build-tool gradle \
  --database postgres \
  --java-orm spring-data-jpa \
  --java-auth spring-security \
  --java-libraries spring-actuator flyway \
  --java-testing-libraries junit5 testcontainers
```

## 创建内容

- Spring Boot Java API。
- Gradle 作为 build tool。
- Spring Data JPA。
- Spring Security。
- Spring Actuator 和 Flyway。
- JUnit 5 和 Testcontainers。

## 生成结构

这个 stack 的起点更接近面向生产的 API baseline。它包含 Gradle wrapper files、Spring Security 配置、JPA 示例 endpoints、Flyway migration files、Actuator endpoints，以及 Testcontainers test path。

```txt
my-secure-spring-api/
├── gradlew
├── gradle/
├── build.gradle.kts
├── settings.gradle.kts
├── src/main/java/com/example/mysecurespringapi/
│   ├── Application.java
│   ├── config/
│   │   └── SecurityConfig.java
│   ├── controller/
│   │   ├── HealthController.java
│   │   └── UserController.java
│   ├── domain/
│   │   └── AppUser.java
│   ├── repository/
│   │   └── AppUserRepository.java
│   └── service/
│       └── AppUserService.java
├── src/main/resources/
│   ├── application.yml
│   └── db/migration/V1__init.sql
└── src/test/java/com/example/mysecurespringapi/
    ├── ApplicationTests.java
    └── ApplicationContainerTests.java
```

生成的 security path 使用 HTTP Basic 和一个 in-memory user。它适合作为本地和内部 API 工作的安全默认值，但不是完整的产品级认证系统。

## 兼容性说明

Spring Security、JPA 和 Spring libraries 需要 Spring Boot 搭配 Java build tool。Flyway 在 Better Fullstack 中与 Spring Data JPA 路径配套。

额外兼容性细节：

- `--java-auth spring-security` 只会为使用 Maven 或 Gradle 的 Spring Boot projects 输出。
- `--java-libraries flyway` 只有在选择 `--java-orm spring-data-jpa` 时保留。
- 如果在更大的 stack 中同时选择 Flyway 和 Liquibase，generator 会在这个路径中保留 Flyway 并丢弃 Liquibase。
- `--java-testing-libraries testcontainers` 会添加基于 container 的 integration test support，因此本地 Docker 可用性会影响这个 test lane。

## 何时选择

适合面向生产的 Java APIs，其中 security、migrations 和 integration tests 应该尽早存在。

当你想要最小 JPA 示例，并计划之后再添加 security 时，选择最小 Spring 指南。当第一个 commit 就需要回答“endpoints 如何保护、schemas 如何变化、如何针对真实数据库测试”时，选择这篇指南。

| 决策点 | Secure Spring API | Minimal Spring JPA |
| --- | --- | --- |
| Security | Spring Security 提供 HTTP Basic | 未选择 |
| Migrations | 从一开始使用 Flyway | 之后添加 |
| Tests | JUnit 5 加 Testcontainers | JUnit 5 |
| Runtime visibility | Actuator health/info/metrics | 基础 health endpoint |
| 成本 | 更多生成文件和配置 | 更小的起点 |

## 代表性片段

生成的 security config 允许访问 `/health`，并用 HTTP Basic 保护应用的其他部分。

```java
@Configuration
public class SecurityConfig {
  @Bean
  SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {
    http
      .csrf((csrf) -> csrf.disable())
      .authorizeHttpRequests((authorize) -> authorize
        .requestMatchers("/health").permitAll()
        .anyRequest().authenticated()
      )
      .httpBasic(Customizer.withDefaults());

    return http.build();
  }
}
```

Credentials 从由环境支持的 properties 中读取，并带有本地 defaults。

```java
@Bean
UserDetailsService userDetailsService(
  @Value("${APP_BASIC_USERNAME:admin}") String username,
  @Value("${APP_BASIC_PASSWORD:change-me}") String password,
  PasswordEncoder passwordEncoder
) {
  UserDetails user = User.withUsername(username)
    .password(passwordEncoder.encode(password))
    .roles("USER")
    .build();

  return new InMemoryUserDetailsManager(user);
}
```

开发时用 Basic auth 访问受保护 endpoints：

```bash
curl -u "$APP_BASIC_USERNAME:$APP_BASIC_PASSWORD" \
  http://localhost:8080/users
```

选择 Flyway 时，schema creation 由 Flyway 负责：

```sql
CREATE TABLE IF NOT EXISTS app_users (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  email VARCHAR(255) NOT NULL UNIQUE,
  display_name VARCHAR(255) NOT NULL,
  created_at TIMESTAMP NOT NULL
);
```

## 迁移与测试

Flyway 会改变 Hibernate 的角色：生成的配置会验证 schema，而不是让 Hibernate 自由更新它。每次数据库变更都添加新的 `V2__...sql` 文件，并让 migrations 保持可 review。

```bash
./gradlew test
./gradlew bootRun
./gradlew build
```

当 Docker 可用时运行 Testcontainers lane。如果本地环境没有 Docker，把 unit tests 与 container tests 分开，让快速 lane 仍然可用。

```java
@SpringBootTest
class ApplicationContainerTests {
  @Test
  void contextLoadsAgainstContainerDatabase() {
  }
}
```

## 部署说明

部署前，替换本地 credentials；如果生成的 build 仍然只包含 H2，添加 PostgreSQL JDBC driver；配置 managed PostgreSQL datasource；并把 Flyway migrations 放进 startup 或 release orchestration 中运行。

```bash
export APP_BASIC_USERNAME="service-admin"
export APP_BASIC_PASSWORD="replace-with-secret"
export SPRING_DATASOURCE_URL="jdbc:postgresql://db.example.com:5432/my_secure_spring_api"
export SPRING_DATASOURCE_DRIVER_CLASS_NAME="org.postgresql.Driver"
export SPRING_DATASOURCE_USERNAME="app"
export SPRING_DATASOURCE_PASSWORD="replace-with-secret"
./gradlew build
java -jar build/libs/*.jar
```

Actuator 通过 `--java-libraries spring-actuator` 选择，因此 health 和 metrics endpoints 可用于平台 probes。在把服务放到公网前，请审查 endpoint exposure。

## 故障排查

| 症状 | 检查 |
| --- | --- |
| 所有 endpoints 都返回 401 | 除 `/health` 外，都需要发送 Basic auth credentials。 |
| Flyway 启动失败 | 确认数据库可访问，并且 migration history 与 schema 匹配。 |
| Testcontainers tests 本地失败 | 确保 Docker 正在运行，且当前用户可以启动 containers。 |
| App 使用默认 `admin/change-me` | 在 runtime environment 中设置 `APP_BASIC_USERNAME` 和 `APP_BASIC_PASSWORD`。 |
| Actuator endpoint 缺失 | 确认 `spring-actuator` 仍然存在于 `--java-libraries`。 |

## 取舍

这个 stack 比最小 Spring Boot API 更重。如果想要更小的起点，使用 [Spring Boot with PostgreSQL and JPA](/guides/java/spring-boot-postgres-jpa/)，或在 builder 中移除可选 libraries。
