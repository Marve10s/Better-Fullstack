Usa este stack cuando quieras una API Java con el ecosistema Spring y persistencia relacional.

```npm
npm create better-fullstack@latest my-spring-api -- \
  --ecosystem java \
  --java-web-framework spring-boot \
  --java-build-tool maven \
  --database postgres \
  --java-orm spring-data-jpa \
  --java-testing-libraries junit5
```

## Qué crea

- Un proyecto Java con Spring Boot.
- Maven como herramienta de build.
- PostgreSQL como opción de base de datos.
- JPA para persistencia.
- JUnit 5 para pruebas.

## Forma generada

El scaffold es un servicio Spring Boot convencional con wrapper scripts, source sets, un controlador de health y endpoints de ejemplo para usuarios respaldados por JPA. El nombre de paquete se deriva del nombre del proyecto bajo `com.example`.

```txt
my-spring-api/
├── mvnw
├── .mvn/
├── pom.xml
├── src/main/java/com/example/myspringapi/
│   ├── Application.java
│   ├── controller/
│   │   ├── HealthController.java
│   │   └── UserController.java
│   ├── domain/
│   │   └── AppUser.java
│   ├── repository/
│   │   └── AppUserRepository.java
│   └── service/
│       └── AppUserService.java
├── src/main/resources/
│   └── application.yml
└── src/test/java/com/example/myspringapi/
    └── ApplicationTests.java
```

El datasource JPA local generado usa una base H2 en archivo con modo de compatibilidad PostgreSQL. Trata `--database postgres` como la selección de base de datos objetivo y cambia la URL del datasource, driver, credenciales y dependencia JDBC para un entorno PostgreSQL real.

## Cuándo elegirlo

Elige Spring Boot cuando quieras un backend Java convencional con un ecosistema grande, integraciones enterprise comunes y una estructura de aplicación predecible.

Elige esta guía cuando quieras una API Spring Boot mínima con persistencia, pero sin seguridad ni tooling de migraciones desde el primer día. Elige la guía segura de Spring cuando todos los endpoints deban empezar detrás de Spring Security y las pruebas de integración con Testcontainers sean parte de la base.

| Punto de decisión | Spring JPA mínimo | API Spring segura |
| --- | --- | --- |
| Build tool | Maven | Gradle |
| Seguridad | No seleccionada | Spring Security seleccionado |
| Cambios de base de datos | Loop local con Hibernate `ddl-auto` | Ruta de migraciones con Flyway |
| Pruebas | JUnit 5 | JUnit 5 más Testcontainers |
| Mejor para | Aprendizaje, prototipos, CRUD interno | Base de API orientada a producción |

## Fragmentos representativos

El modelo de dominio generado es una entidad JPA estándar. Agrega campos aquí cuando pertenezcan al aggregate persistido.

```java
import jakarta.persistence.Entity;
import jakarta.persistence.GeneratedValue;
import jakarta.persistence.GenerationType;
import jakarta.persistence.Id;
import java.time.Instant;

@Entity
public class AppUser {
  @Id
  @GeneratedValue(strategy = GenerationType.IDENTITY)
  private Long id;

  private String email;
  private String displayName;
  private Instant createdAt = Instant.now();

  protected AppUser() {
  }

  public AppUser(String email, String displayName) {
    this.email = email;
    this.displayName = displayName;
  }
}
```

Los repositories se mantienen pequeños porque Spring Data JPA deriva consultas comunes a partir de nombres de métodos.

```java
import java.util.Optional;
import org.springframework.data.jpa.repository.JpaRepository;

public interface AppUserRepository extends JpaRepository<AppUser, Long> {
  Optional<AppUser> findByEmail(String email);
}
```

El controller expone `GET /users` y `POST /users` a través de la capa de servicios.

```java
@RestController
@RequestMapping("/users")
public class UserController {
  private final AppUserService userService;

  public UserController(AppUserService userService) {
    this.userService = userService;
  }

  @GetMapping
  public List<UserResponse> listUsers() {
    return userService.findAll().stream().map(UserResponse::from).toList();
  }

  @PostMapping
  @ResponseStatus(HttpStatus.CREATED)
  public UserResponse createUser(@RequestBody CreateUserRequest request) {
    AppUser createdUser = userService.create(request.email(), request.displayName());
    return UserResponse.from(createdUser);
  }
}
```

## Migraciones y pruebas

Este stack mínimo no selecciona Flyway ni Liquibase. Hibernate puede mantener el schema local de H2 avanzando durante el desarrollo temprano, pero agrega una librería de migraciones antes de que varios entornos dependan de la forma de la base de datos.

```bash
./mvnw test
./mvnw spring-boot:run
./mvnw package
```

Cuándo agregues PostgreSQL de producción, pasa de defaults locales a settings de datasource guiados por entorno:

```yaml
spring:
  datasource:
    url: ${DATABASE_URL}
    username: ${DATABASE_USERNAME}
    password: ${DATABASE_PASSWORD}
  jpa:
    hibernate:
      ddl-auto: validate
```

## Notas de compatibilidad

- `--java-orm spring-data-jpa` requiere Spring Boot y una herramienta de build Java.
- Flyway y Liquibase están vinculados al camino de Spring Data JPA en Better Fullstack.
- Esta guía usa `--java-build-tool maven`; usa los comandos del Maven wrapper salvo que regeneres intencionalmente con Gradle.
- El datasource y la dependencia locales generados son H2 aunque este stack apunte a PostgreSQL. Agrega el driver JDBC de PostgreSQL y sobrescribe `spring.datasource.*` antes de desplegar contra una base Postgres administrada.

## Notas de despliegue

Construye el jar con Maven, proporciona Java 21 o superior y ejecuta la aplicación con settings de datasource específicos del entorno.

```bash
./mvnw package
java -jar target/*.jar
```

Para producción, agrega migraciones, agrega el driver JDBC de PostgreSQL, configura connection pooling mediante propiedades de datasource de Spring Boot y evita depender de `ddl-auto: update`. Expón `/health` mediante el health check de tu plataforma.

## Troubleshooting

| Síntoma | Revisa |
| --- | --- |
| La app arranca con H2 en vez de Postgres | Sobrescribe `spring.datasource.*` en la configuración específica del entorno. |
| `POST /users` acepta input incompleto | Agrega `--java-libraries spring-validation` en un scaffold futuro o Bean Validation manualmente. |
| Las entidades JPA no aparecen | Mantén las entidades bajo el paquete de aplicación generado para que component scanning las encuentre. |
| Fallan los comandos Maven | Usa `./mvnw`, no un Maven global con otra versión. |

## Tradeoffs

Spring Boot es más pesado que frameworks de API mínimos, pero compensa cuando necesitas convenciones establecidas, profundidad de librerías y mantenibilidad de backend a largo plazo.
