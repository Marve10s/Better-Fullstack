当你想要围绕 TanStack Router 和 TanStack Start server route model 构建 React fullstack app 时，使用这个 stack。

```npm
npm create better-fullstack@latest my-tanstack-app -- \
  --ecosystem typescript \
  --frontend tanstack-start \
  --backend self \
  --database sqlite \
  --orm drizzle \
  --auth better-auth \
  --api trpc \
  --css-framework tailwind \
  --ui-library shadcn-ui \
```

## 创建内容

- 带 file-based routes 的 TanStack Start web app。
- 通过 framework 内置 server layer 实现 fullstack routing。
- SQLite 和 Drizzle，用于小型本地关系型数据库设置。
- 已接入 Better Auth。
- frontend 使用 Tailwind CSS 和 shadcn/ui。
- `bts.jsonc` 中保存可复现的 stack 配置。

## 生成结构

生成的项目是一个单独可部署的 web app，而不是拆开的 frontend/API workspace。TanStack Start 负责 routing、server functions 和 request handling；Drizzle 负责 database schema 和 queries。

代表性结构：

```txt
my-tanstack-app/
  bts.jsonc
  package.json
  src/
    routes/
    components/
    lib/
      auth/
      db/
      trpc/
```

准确文件名可能随 generator 演进，但边界保持稳定：route code 留在 TanStack Start app 中，database code 放在 Drizzle helpers 后面，API procedures 通过 `--api trpc` 暴露。

## Route 和 API 模式

典型 route 会把 UI state 留在 React 中，并调用 typed API helper，而不是手写 `fetch` URL：

```tsx
import { createFileRoute } from "@tanstack/react-router";

export const Route = createFileRoute("/dashboard")({
  component: DashboardPage,
});

function DashboardPage() {
  return (
    <main className="mx-auto max-w-5xl p-6">
      <h1 className="text-2xl font-semibold">Dashboard</h1>
      <p className="text-muted-foreground">
        Load account data through the generated server API layer.
      </p>
    </main>
  );
}
```

新增 API 时，应围绕 use case 建立窄 procedures，而不是一开始就暴露表形 CRUD：

```ts
import { z } from "zod";

export const projectInput = z.object({
  name: z.string().min(2),
});

export async function createProject(input: z.infer<typeof projectInput>) {
  const values = projectInput.parse(input);

  return {
    id: crypto.randomUUID(),
    name: values.name,
  };
}
```

## Database model 示例

Drizzle 在 schema 小而明确时效果最好。一个代表性的 SQLite table 可能像这样：

```ts
import { integer, sqliteTable, text } from "drizzle-orm/sqlite-core";

export const project = sqliteTable("project", {
  id: text("id").primaryKey(),
  name: text("name").notNull(),
  ownerId: text("owner_id").notNull(),
  createdAt: integer("created_at", { mode: "timestamp" }).notNull(),
});
```

把 auth 拥有的 tables 与 product tables 分开。Better Auth 可能会根据所选 adapter 生成或期望自己的 schema；product models 应引用 user identifier，而不是复制 auth state。

## 何时选择

当你需要 React、typed routing、colocated fullstack behavior，并且想要比大型 batteries-included framework 更小的 framework surface 时，选择 TanStack Start。

它适合 dashboards、internal tools、SaaS prototypes，以及 route type safety 很重要的应用。

## 部署

部署前，确保为 database、Better Auth secrets，以及 auth callbacks 使用的 public application URL 配好环境变量。对于 SQLite 部署，请确认 host 支持持久化可写存储；许多 serverless platforms 不支持。

如果目标是 serverless platform，PostgreSQL 通常比本地 SQLite 文件更适合作为生产数据库。提交 `bts.jsonc`，方便之后的 teammates 和 agents 还原预期 stack。

## 故障排查

- 如果 auth callbacks 重定向到错误 host，先检查 app base URL 和 Better Auth 环境变量。
- 如果 database queries 在本地正常、部署后失败，请确认 production database URL，并检查 migrations 是否已在部署的 database 上运行。
- 如果 generated route types 看起来过期，重启 framework type generation process，并重新运行最小相关 typecheck。
- 如果 tRPC client calls 返回 404，请确认 generated API route 仍然挂载在 client 期望的位置。

## Tradeoffs

当你想深入使用 TanStack ecosystem 时，TanStack Start 更合适。如果你想要最大的 hosting 和 documentation ecosystem，可以与 [Next.js with Drizzle and Better Auth](/guides/typescript/nextjs-drizzle-better-auth/) 对比。

与单独的 Hono backend 相比，这个 stack 的 deployable services 更少，cross-origin 设置也更少。与 Next.js 相比，它提供 TanStack Router-first 的心智模型和更小的 framework footprint，但生态系统也更年轻。
