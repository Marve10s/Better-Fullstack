Usa este stack cuando quieras un proyecto Next.js preparado para flujos de desarrollo asistidos por IA, herramientas MCP locales y documentación generada para agentes.

```npm
npm create better-fullstack@latest my-agent-workbench -- \
  --ecosystem typescript \
  --frontend next \
  --backend self \
  --database sqlite \
  --orm drizzle \
  --auth better-auth \
  --api trpc \
  --ai ai-cli \
  --addons mcp skills \
  --ai-docs claude-md agents-md cursorrules \
  --css-framework tailwind \
  --ui-library shadcn-ui \
```

## Qué crea

- Una app Next.js con setup fullstack local.
- SQLite y Drizzle.
- Better Auth.
- Configuración de AI CLI.
- Add-ons MCP y Skills.
- Archivos de documentación para agentes como `CLAUDE.md`, `AGENTS.md` o `.cursorrules`, según los flags seleccionados.

## Forma generada

Este preset genera tanto la aplicación como el flujo de trabajo para agentes. La app web es un proyecto Next.js fullstack normal, mientras que los add-ons crean contexto de repositorio para agentes de código y herramientas locales de IA.

Forma representativa:

```txt
my-agent-workbench/
  bts.jsonc
  AGENTS.md
  CLAUDE.md
  .cursorrules
  app/
  components/
  lib/
    auth/
    db/
```

Los archivos de documentación exactos dependen de `--ai-docs claude-md agents-md cursorrules`. El comando mostrado pide los tres para que distintas herramientas de agentes encuentren guía del proyecto en su formato preferido.

## Cómo encajan las piezas AI

`--ai ai-cli` selecciona soporte para AI CLI. `--addons mcp skills` agrega capacidades locales de agentes alrededor de MCP y skills. `--ai-docs` controla qué archivos de instrucciones del repositorio se generan.

Usa esas piezas para trabajos distintos:

| Pieza | Propósito | Uso típico |
| --- | --- | --- |
| AI CLI | Flujo local de IA en línea de comandos | Ejecutar tareas de proyecto asistidas por agentes |
| MCP | Puente de herramientas/contexto | Exponer capacidades del proyecto o servicios a agentes |
| Skills | Instrucciones reutilizables para agentes | Estandarizar flujos como debugging o trabajo de features |
| Agent docs | Guía del repositorio | Explicar a los agentes cómo está estructurado el proyecto |

## Instrucción de agente de ejemplo

La documentación para agentes debe ser concreta. Prefiere reglas de proyecto que se puedan seguir y verificar:

```md
## Verification

After editing authentication, run the smallest auth-related test or typecheck.
Do not run broad workspace commands unless the change crosses package boundaries.

## Project boundaries

Database access belongs in server-only modules.
Client components should call typed API helpers instead of importing Drizzle code.
```

Mantén estos docs al día cuando cambie la arquitectura. Las instrucciones obsoletas para agentes son peores que no tener instrucciones, porque dirigen el trabajo futuro con confianza hacia archivos equivocados.

## Modelo de aplicación de ejemplo

El lado app sigue siendo un stack TypeScript normal de Better Fullstack. Una tabla SQLite pequeña podría verse así:

```ts
import { integer, sqliteTable, text } from "drizzle-orm/sqlite-core";

export const agentRun = sqliteTable("agent_run", {
  id: text("id").primaryKey(),
  goal: text("goal").notNull(),
  status: text("status").notNull(),
  createdAt: integer("created_at", { mode: "timestamp" }).notNull(),
});
```

Usa tablas de aplicación para datos de producto, no para almacenar secretos. Credenciales MCP, tokens de proveedores y claves de servicios AI deben vivir en variables de entorno o almacenamiento seguro específico del proveedor.

## Cuándo elegirlo

Elige esto cuando el propio repositorio deba ser fácil de entender y modificar para agentes de código. Es útil para equipos que estandarizan flujos de agentes, herramientas MCP y contexto generado de proyecto.

## Compatibilidad

- Este preset usa una app TypeScript Next.js con `--backend self`, así que Next.js es dueño del límite de servidor.
- `--addons mcp skills` son adiciones de flujo de desarrollo. No reemplazan las decisiones de auth, base de datos ni API de la app.
- `--ai-docs claude-md agents-md cursorrules` genera intencionalmente varios formatos de instrucciones para distintas herramientas.
- SQLite es conveniente para proyectos tipo workbench locales. Usa PostgreSQL si el workbench se convierte en un producto multiusuario de producción con estado compartido.

## Despliegue

Despliega la app Next.js como una aplicación fullstack normal: configura secretos de Better Auth, app URL y ajustes de base de datos. Trata claves de proveedores AI, credenciales MCP y cualquier token de herramientas como secretos server-side.

Los docs de agentes normalmente pertenecen al control de versiones. Credenciales locales, logs generados y configuración MCP específica de una máquina normalmente no. Revisa los archivos ignore generados antes de commitear.

## Solución de problemas

- Si una herramienta AI ignora la guía del repositorio, confirma que lee el filename generado que seleccionaste con `--ai-docs`.
- Si las herramientas MCP no están disponibles, verifica el setup MCP local y si el agente consumidor apunta a él.
- Si el setup de add-ons hace prompts durante ejecuciones scriptadas, usa flags explícitos como en esta guía para que los defaults sean deterministas.
- Si la app funciona localmente pero auth falla tras desplegar, revisa los mismos settings de URL y cookies de Better Auth que revisarías en cualquier stack Next.js.

## Tradeoffs

Este stack es más opinado que un starter Next.js mínimo. Si solo necesitas una app de producto sin tooling de agentes, usa [Next.js con Drizzle y Better Auth](/guides/typescript/nextjs-drizzle-better-auth/).

Los archivos extra son valiosos cuando humanos y agentes trabajan repetidamente en el repo. Son overhead para un prototipo descartable. Si el equipo no va a mantener las instrucciones, empieza con la guía Next.js más simple y agrega AI docs después.
