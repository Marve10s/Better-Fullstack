当你想要 Go 终端 UI，而不是 web API 时，使用这个 stack。

```npm
npm create better-fullstack@latest my-go-cli -- \
  --ecosystem go \
  --go-web-framework none \
  --go-orm none \
  --go-api none \
  --go-cli bubbletea \
  --go-logging none \
  --go-auth none
```

## 创建内容

- Go CLI 项目。
- Bubble Tea 作为终端 UI 选项。
- 不包含 web framework、ORM、gRPC 或 auth add-on。

## 生成结构

这个 stack 创建的是交互式终端应用，不是 server。Bubble Tea 会围绕 model、message、update 和 view 组织程序。

主循环很简单：

- model 保存当前应用状态。
- message 描述 input、tick 或 command result。
- `Update` 修改 model 并返回后续 command。
- `View` 将当前 model 渲染为终端文本。

## 代表性文件树

```txt
my-go-cli/
  go.mod
  cmd/
    app/
      main.go
  internal/
    tui/
      model.go
      update.go
      view.go
```

尽量把副作用放进 Bubble Tea command。这样 model 和 view 更容易在不运行真实终端的情况下测试。

## Model 与 update 示例

Bubble Tea model 可以先只包含足够渲染和响应输入的状态：

```go
package tui

type Model struct {
	Items    []string
	Selected int
	Quitting bool
}

func NewModel() Model {
	return Model{
		Items: []string{"Build", "Test", "Deploy"},
	}
}
```

update 函数负责状态转换：

```go
package tui

import tea "github.com/charmbracelet/bubbletea"

func (m Model) Update(msg tea.Msg) (tea.Model, tea.Cmd) {
	switch msg := msg.(type) {
	case tea.KeyMsg:
		switch msg.String() {
		case "q", "ctrl+c":
			m.Quitting = true
			return m, tea.Quit
		case "j", "down":
			if len(m.Items) > 0 {
				m.Selected = (m.Selected + 1) % len(m.Items)
			}
		case "k", "up":
			if len(m.Items) > 0 {
				m.Selected = (m.Selected - 1 + len(m.Items)) % len(m.Items)
			}
		}
	}

	return m, nil
}
```

view 应该只根据状态渲染，不执行额外工作：

```go
package tui

import "strings"

func (m Model) View() string {
	if m.Quitting {
		return ""
	}

	var b strings.Builder
	b.WriteString("Tasks\n\n")

	for i, item := range m.Items {
		cursor := " "
		if i == m.Selected {
			cursor = ">"
		}
		b.WriteString(cursor + " " + item + "\n")
	}

	b.WriteString("\nq: quit  j/k: move\n")
	return b.String()
}
```

## 兼容性说明

CLI 项目应保持 server-only category 为 `none`，除非你有意构建 hybrid tool。

为 terminal-first app 保持 `--go-cli bubbletea`、`--go-web-framework none`、`--go-orm none` 和 `--go-api none`。如果 CLI 之后要调用 API，请把 client 放在独立 package 中，并通过 command 调用，不要从 `View` 调用。

## 何时选择

当你要构建终端 dashboard、开发者工具，或需要交互体验的命令行界面时，选择 Bubble Tea。

它很适合需要选择、进度、表单或实时状态，但又不想构建浏览器 UI 的工具。

## 测试与分发

直接测试 model transition：

```bash
go test ./...
```

分发时构建单个 binary，并记录支持的 environment variable 或 config file：

```bash
go build ./cmd/app
```

如果生成项目使用不同的 command 路径，请以它的 `cmd/` 目录为准。

## 故障排查

- 如果按键没有反应，确认当前聚焦的终端发送的是 Bubble Tea 能收到的 key sequence。
- 如果退出后屏幕残留旧输出，检查程序启动和关闭时的 alternate-screen 选项。
- 如果测试难写，把副作用移动到 `tea.Cmd` 函数中，并保持 `Update` 可预测。
- 如果输出换行很差，用窄宽度测试，并在 model 跟踪尺寸时让 `View` 的格式适配可用空间。
