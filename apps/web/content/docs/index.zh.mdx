Better Fullstack 是面向类型化产品 stack 的项目生成器。选择 frontend、backend、数据层、服务、tooling 和目标生态；CLI 会写入匹配的文件，并把 stack 记录到 `bts.jsonc`。

需要引导式配置时使用交互式 CLI；想可视化比较选项时使用 [Stack Builder](https://better-fullstack.dev/new)；需要给 CI、文档或其他 developer 留下可复现命令时使用显式 flags。

## 要求

CLI 运行在 Node.js 20 或更高版本。

```bash
node --version
```

生成的项目也可能需要对应生态的 toolchain：React Native 需要 Expo tooling，Rust 需要 Cargo，Python 需要 `uv`，Go 需要 Go，Java 需要 Java 21，Phoenix 项目需要 Elixir/Erlang。

## 创建第一个项目

运行向导并回答 prompts：

```npm
npm create better-fullstack@latest my-app
```

使用 npm 时，Better Fullstack flags 要放在 `--` 分隔符之后。package-manager tabs 会展示对应的 pnpm、Bun 和 Yarn 命令，这些命令不需要 npm 专属的分隔符。

## 默认配置会创建什么

接受 defaults 会创建一个 TypeScript 项目，包含：

- TanStack Router frontend 和 Hono backend。
- Bun runtime、SQLite、Drizzle、Better Auth 和 tRPC。
- Tailwind CSS、shadcn/ui、Vitest 和 Turborepo。
- 启动 CLI 时使用的 package manager、稳定依赖版本、Git、启用安装，以及 `CLAUDE.md`。

你可以保留这个 stack，从模板开始，或完全切换到其他生态。

## 选择 stack 的方式

| 路径 | 适合场景 |
| --- | --- |
| CLI 向导 | 你想要 prompts 和合理 defaults。 |
| Stack Builder | 你想在生成前探索兼容组合。 |
| 显式 flags | 你想在 scripts、docs 或 issue reports 中得到可重复 output。 |
| Agent Skill | 你想让 coding agent 使用 CLI workflow，而不是手写 starter 文件。 |
| MCP server | 你想让 AI coding agent 检查 schema、验证兼容性，并安全地 dry-run。 |

## 生成项目的上下文

每个生成项目都会包含：

- 记录所选 stack 的 `bts.jsonc`。
- CLI 打印的可复现命令。
- 面向所选生态的 framework、package 和 config 文件。
- 可选的 AI 指令文件，例如 `CLAUDE.md`、`Agents.md` 或 `.cursorrules`。
