title: Primer proyecto
description: Genera, inspecciona, instala y ejecuta un proyecto Better Fullstack.
---

Este walkthrough crea un proyecto, previsualiza lo que la CLI puede generar y muestra donde mirar después de que termina el scaffold.

## 1. Crea un proyecto

Ejecuta el asistente:

```npm
npm create better-fullstack@latest my-app
```

Acepta los defaults para un starter TypeScript o cambia los prompts para ajustarlos al stack que quieres.

## 2. Previsualiza sin escribir

Usa `--dry-run` cuando quieras inspeccionar primero el árbol de archivos generado:

```npm
npm create better-fullstack@latest my-app -- --dry-run
```

El dry run es util antes de probar un ecosistema nuevo, un target de deployment o un stack multi-ecosystem.

## 3. Genera desde un comando

Los flags explicitos hacen que el scaffold sea repetible. Este ejemplo crea un proyecto Next.js con self-backend y salta install/Git para que puedas revisar los archivos primero:

```npm
npm create better-fullstack@latest my-app -- \
  --ecosystem typescript \
  --frontend next \
  --backend self \
  --runtime none \
  --database postgres \
  --orm drizzle \
  --auth better-auth \
  --api trpc \
  --db-setup none \
  --ai none \
  --payments none \
  --examples none \
  --addons none \
  --ai-docs none \
  --version-channel stable \
  --no-install \
  --no-git
```

<Callout kind="info" title="Comportamiento de prompts">
Los flags explicitos saltan las categorías que cubren. `--yes` acepta defaults y entra en conflicto con flags centrales del stack como `--frontend`, `--backend` y `--database`. Usa `none` para categorías opcionales que quieras desactivar.
</Callout>

## 4. Instala dependencias

Si usaste `--no-install`, instala manualmente dentro del proyecto:

```bash
cd my-app
```

```npm
npm install
```

Los ecosistemas nativos de lenguaje usan sus propias herramientas:

```bash
cargo build
uv sync
go mod tidy
./mvnw test
./gradlew test
```

## 5. Inspecciona el proyecto

Los proyectos generados incluyen:

- `bts.jsonc` con el stack seleccionado y la referencia del schema.
- Archivos de framework, package y config para el ecosistema elegido.
- Un comando reproducible impreso por la CLI.
- Docs opcionales para IA como `CLAUDE.md`, `Agents.md` o `.cursorrules`.
- README y notas de setup cuando las plantillas seleccionadas las proveen.

## 6. Agrega features soportadas después

Dentro de un proyecto generado, usa `add` para addons soportados:

```npm
npm create better-fullstack@latest add -- --addons mcp skills
```

El comando `add` lee `bts.jsonc`, omite addons ya presentes e imprime guía de instalacion cuando las dependencias no se instalan automaticamente.
