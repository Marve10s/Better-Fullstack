Better Fullstack se ejecuta mediante el paquete `create-better-fullstack`. No necesitas instalarlo globalmente; ejecutalo con tu package manager, responde los prompts y la CLI escribira el proyecto seleccionado.

<Callout kind="tip" title="Preview online">
Prefieres elegir visualmente? Abre el [Stack Builder](https://better-fullstack.dev/new), elige un stack compatible y copia el comando generado.
</Callout>

## Prerrequisitos

- Node.js 20 o superior para la CLI.
- Una terminal.
- Git solo si quieres que la CLI inicialice un repositorio.
- Herramientas específicas del ecosistema solo para el proyecto que generes.

| Ecosistema | Tooling extra |
| --- | --- |
| TypeScript | npm, pnpm, Yarn o Bun |
| React Native | Tooling iOS/Android compatible con Expo cuando corras en simulador o dispositivo |
| Rust | Rust toolchain y Cargo |
| Python | Python y `uv` |
| Go | Go toolchain |
| Java | Java 21 |
| Elixir | Elixir, Erlang/OTP y PostgreSQL para flujos Phoenix con base de datos |

## Ejecuta la CLI

<PMTabs
  npm="npm create better-fullstack@latest"
  pnpm="pnpm create better-fullstack@latest"
  bun="bun create better-fullstack@latest"
  yarn="yarn create better-fullstack@latest"
/>

Agrega un nombre de proyecto para iniciar el asistente:

```npm
npm create better-fullstack@latest my-app
```

El asistente pregunta por ecosistema, opciones de stack, addons opcionales, Git e instalacion de dependencias.

## Usa flags explicitos

Los flags explicitos saltan los prompts que cubren. Es el mejor formato para scripts, docs, comandos generados y bug reports reproducibles.

```npm
npm create better-fullstack@latest my-app -- \
  --ecosystem typescript \
  --frontend tanstack-router \
  --backend hono \
  --runtime node \
  --database sqlite \
  --orm drizzle \
  --api orpc \
  --auth better-auth \
  --db-setup none \
  --ai-docs none \
  --version-channel stable \
  --no-install \
  --no-git
```

Con `npm create`, los flags de Better Fullstack van después de `--`. Las tabs renderizadas para pnpm, Bun y Yarn eliminan ese separador exclusivo de npm.

Usa `--ai-docs none` cuando no quieras generar archivos de instrucciones para IA. Usa `--no-install` cuando quieras inspeccionar los archivos antes de instalar dependencias.

## Plantillas

Las plantillas son atajos para stacks comunes:

| Plantilla | Stack |
| --- | --- |
| `t3` | Next.js, tRPC, Prisma, PostgreSQL, Better Auth |
| `mern` | MongoDB, Express, React Router, Node.js |
| `pern` | PostgreSQL, Express, TanStack Router, Node.js |
| `uniwind` | Expo + Uniwind, solo mobile |
| `none` | Sin plantilla abreviada |

```npm
npm create better-fullstack@latest my-app -- --template t3
```

## Valida antes de escribir archivos

Usa `--dry-run` para previsualizar archivos generados sin escribir un proyecto:

```npm
npm create better-fullstack@latest my-app -- --dry-run
```

Usa `--verify` cuando quieras que la CLI ejecute checks generados después del scaffold. No inicia dev servers.
