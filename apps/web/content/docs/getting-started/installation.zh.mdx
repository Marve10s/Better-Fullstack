Better Fullstack 通过 `create-better-fullstack` 包运行。你不需要全局安装它；用你的 package manager 执行，回答 prompts，CLI 就会写入所选项目。

<Callout kind="tip" title="在线预览">
更想可视化选择？打开 [Stack Builder](https://better-fullstack.dev/new)，选择兼容 stack，然后复制生成的命令。
</Callout>

## 前置条件

- CLI 需要 Node.js 20 或更高版本。
- 一个终端。
- 只有在你希望 CLI 初始化仓库时才需要 Git。
- 只为你要生成的项目安装对应生态工具。

| 生态系统 | 额外 tooling |
| --- | --- |
| TypeScript | npm、pnpm、Yarn 或 Bun |
| React Native | 在模拟器或设备上运行时，需要 Expo 兼容的 iOS/Android tooling |
| Rust | Rust toolchain 和 Cargo |
| Python | Python 和 `uv` |
| Go | Go toolchain |
| Java | Java 21 |
| Elixir | 数据库驱动的 Phoenix flows 需要 Elixir、Erlang/OTP 和 PostgreSQL |

## 运行 CLI

<PMTabs
  npm="npm create better-fullstack@latest"
  pnpm="pnpm create better-fullstack@latest"
  bun="bun create better-fullstack@latest"
  yarn="yarn create better-fullstack@latest"
/>

添加项目名来启动向导：

```npm
npm create better-fullstack@latest my-app
```

向导会询问生态系统、stack 选项、可选 addons、Git 和依赖安装。

## 使用显式 flags

显式 flags 会跳过它们覆盖的 prompts。这最适合 scripts、docs、生成命令和可复现 bug reports。

```npm
npm create better-fullstack@latest my-app -- \
  --ecosystem typescript \
  --frontend tanstack-router \
  --backend hono \
  --runtime node \
  --database sqlite \
  --orm drizzle \
  --api orpc \
  --auth better-auth \
  --db-setup none \
  --ai-docs none \
  --version-channel stable \
  --no-install \
  --no-git
```

使用 `npm create` 时，Better Fullstack flags 要放在 `--` 之后。渲染后的 pnpm、Bun 和 Yarn tabs 会移除这个 npm 专属分隔符。

不想生成 AI 指令文件时使用 `--ai-docs none`。想在安装依赖前先检查文件时使用 `--no-install`。

## 模板

模板是常见 stack 的简写：

| 模板 | Stack |
| --- | --- |
| `t3` | Next.js、tRPC、Prisma、PostgreSQL、Better Auth |
| `mern` | MongoDB、Express、React Router、Node.js |
| `pern` | PostgreSQL、Express、TanStack Router、Node.js |
| `uniwind` | Expo + Uniwind，仅 mobile |
| `none` | 不使用简写模板 |

```npm
npm create better-fullstack@latest my-app -- --template t3
```

## 写入文件前先验证

使用 `--dry-run` 预览生成文件，不写入项目：

```npm
npm create better-fullstack@latest my-app -- --dry-run
```

当你希望 CLI 在 scaffold 后运行生成的 checks 时使用 `--verify`。它不会启动 dev servers。
