---
title: 第一个项目
description: Scaffold、检查、安装并运行一个 Better Fullstack 项目。
---

这个 walkthrough 会创建一个项目，预览 CLI 可以生成什么，并说明 scaffold 完成后应该检查哪里。

## 1. 创建项目

运行向导：

```npm
npm create better-fullstack@latest my-app
```

接受 TypeScript starter 的 defaults，或调整 prompts 来匹配你想要的 stack。

## 2. 不写入文件先预览

想先检查生成的文件树时使用 `--dry-run`：

```npm
npm create better-fullstack@latest my-app -- --dry-run
```

在尝试新生态、新 deployment target 或 multi-ecosystem stack 前，dry run 很有用。

## 3. 从命令生成

显式 flags 让 scaffold 可重复。这个例子创建一个 Next.js self-backend 项目，并跳过 install/Git，方便你先审查文件：

```npm
npm create better-fullstack@latest my-app -- \
  --ecosystem typescript \
  --frontend next \
  --backend self \
  --runtime none \
  --database postgres \
  --orm drizzle \
  --auth better-auth \
  --api trpc \
  --db-setup none \
  --ai none \
  --payments none \
  --examples none \
  --addons none \
  --ai-docs none \
  --version-channel stable \
  --no-install \
  --no-git
```

<Callout kind="info" title="Prompt 行为">
显式 flags 会跳过它们覆盖的 categories。`--yes` 接受 defaults，并与 `--frontend`、`--backend`、`--database` 等核心 stack flags 冲突。对想关闭的可选 categories 使用 `none`。
</Callout>

## 4. 安装依赖

如果你使用了 `--no-install`，在项目内手动安装：

```bash
cd my-app
```

```npm
npm install
```

语言原生生态使用自己的工具：

```bash
cargo build
uv sync
go mod tidy
./mvnw test
./gradlew test
```

## 5. 检查项目

生成项目包含：

- 记录所选 stack 和 schema reference 的 `bts.jsonc`。
- 面向所选生态的 framework、package 和 config 文件。
- CLI 打印的可复现命令。
- 可选 AI docs，例如 `CLAUDE.md`、`Agents.md` 或 `.cursorrules`。
- 当所选模板提供时，会包含 README 和 setup notes。

## 6. 后续添加支持的 features

在生成项目内，使用 `add` 添加支持的 addons：

```npm
npm create better-fullstack@latest add -- --addons mcp skills
```

`add` 命令会读取 `bts.jsonc`，跳过已经存在的 addons，并在依赖未自动安装时打印安装指引。
