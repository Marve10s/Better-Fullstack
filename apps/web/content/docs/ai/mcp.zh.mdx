title: MCP 服务器
description: 将 Better Fullstack 连接到 Claude Code、Cursor、VS Code 和兼容 MCP 的 AI 代码代理。
---

Better Fullstack MCP server 让 AI 代理通过结构化工具调用来生成项目，而不是猜测 CLI 参数。它负责查询 schema、验证兼容性、预览生成结果、创建项目，以及为现有项目添加功能。

此页面介绍 Better Fullstack 的 MCP server。生成项目时选择的 `mcp` addon 是另一件事：它会把 MCP client/server 配置写入生成出来的应用，并可能运行外部设置命令。

如果想使用更轻量的 CLI 驱动流程，请安装 [Better Fullstack agent skill](/docs/ai/skills/)。skill 和 MCP server 使用同一个生成器：skill 更适合快速的参数式脚手架流程，MCP 则更适合结构化工具调用。

## 设置

使用你偏好的包管理器启动 server：

<PMTabs
  npm="npx -y create-better-fullstack@latest mcp"
  pnpm="pnpm dlx create-better-fullstack@latest mcp"
  bun="bunx create-better-fullstack@latest mcp"
  yarn="yarn dlx create-better-fullstack@latest mcp"
/>

### Claude Code

```bash
claude mcp add --transport stdio better-fullstack -- npx -y create-better-fullstack@latest mcp
```

如果你更喜欢 pnpm、Bun 或 Yarn，请使用上方同一启动方式表。

### Cursor

```json
{
  "mcpServers": {
    "better-fullstack": {
      "command": "npx",
      "args": ["-y", "create-better-fullstack@latest", "mcp"]
    }
  }
}
```

### VS Code

```json
{
  "servers": {
    "better-fullstack": {
      "type": "stdio",
      "command": "npx",
      "args": ["-y", "create-better-fullstack@latest", "mcp"]
    }
  }
}
```

## 工具

完整的输入、输出、示例和安全写入行为，请参阅 [MCP tools reference](/docs/ai/mcp-tools/)。

| 工具 | 用途 |
| --- | --- |
| `bfs_get_guidance` | 返回工作流规则和字段语义。先调用它。 |
| `bfs_get_schema` | 返回配置分类的有效选项。 |
| `bfs_check_compatibility` | 验证 stack，并返回包含 warnings 的调整后 selections。 |
| `bfs_plan_project` | 在内存中 dry-run 项目生成结果。 |
| `bfs_create_project` | 将新项目写入磁盘。 |
| `bfs_plan_addition` | 验证现有项目的 addon 变更。 |
| `bfs_add_feature` | 向现有项目添加受支持的 addons。 |

<Callout kind="info" title="安装行为">
MCP 创建项目时会跳过普通依赖安装，以避免长时间的代理调用。如果生成项目包含 `mcp` addon，该 addon 的设置流程仍可能为了 MCP 客户端配置而运行外部命令。
</Callout>

## 资源

| Resource | 内容 |
| --- | --- |
| `docs://compatibility-rules` | stack 组合的兼容性规则。 |

## 推荐工具顺序

新项目：

1. `bfs_get_guidance`
2. `bfs_get_schema`
3. `bfs_check_compatibility`
4. `bfs_plan_project`
5. `bfs_create_project`

现有项目：

1. `bfs_plan_addition`
2. `bfs_add_feature`

## Agents 关键规则

- `frontend` 是 array，可以包含多个 frontend targets。
- `none` 会禁用一个分类。
- 先设置 `ecosystem`，这样 agent 会使用正确的字段族。
- Array fields 包括 `frontend`、`addons`、`examples`、`rustLibraries`、`pythonAi`、`javaLibraries` 和 `javaTestingLibraries`。
- Project planning、creation 和 addition responses 包含 `graphSummary`、`effectiveStack` 和 `stackPartSpecs`，用于 graph-backed stack view。
- 当前 MCP project schemas 不把 `aiDocs` 暴露为 create field；project creation 使用 server 内置的 AI-doc behavior。
- 创建项目前始终调用 `bfs_check_compatibility`。

## Agent prompt

```text
Use Better Fullstack MCP. First call bfs_get_guidance to understand
the rules, then bfs_get_schema for the relevant categories, then
bfs_check_compatibility to validate the stack, then bfs_plan_project
for a dry-run. Only call bfs_create_project after the dry-run succeeds.
Skip normal dependency installation and tell me the exact commands to
install, test, and run the project.
```
