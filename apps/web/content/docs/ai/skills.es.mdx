La skill de agente de Better Fullstack enseña a los agentes de código a usar la CLI en lugar de escribir proyectos starter a mano. Mapea el stack solicitado a partes de grafo, ejecuta primero un dry-run, genera el scaffold con la instalación de dependencias deshabilitada y reporta los comandos exactos de seguimiento.

Usa esto cuando tu agente soporte skills y quieras la ruta más rápida desde un prompt hasta un proyecto Better Fullstack generado. Usa el [servidor MCP](/docs/ai/mcp/) cuando quieras búsqueda estructurada de schema y planificación con tool-calls.

## Instalar

La skill vive en este repositorio en `.agents/skills/better-fullstack`. Cómo el repo también contiene otras skills internas, instala solo `better-fullstack`.

Instalar para Codex:

<PMTabs
  npm="npx skills@latest add Marve10s/Better-Fullstack --skill better-fullstack --agent codex --global --yes"
  pnpm="pnpm dlx skills@latest add Marve10s/Better-Fullstack --skill better-fullstack --agent codex --global --yes"
  bun="bunx skills@latest add Marve10s/Better-Fullstack --skill better-fullstack --agent codex --global --yes"
  yarn="yarn dlx skills@latest add Marve10s/Better-Fullstack --skill better-fullstack --agent codex --global --yes"
/>

Instalar para varios agentes comunes:

```bash
bunx skills@latest add Marve10s/Better-Fullstack \
  --skill better-fullstack \
  --agent codex claude-code cursor github-copilot gemini-cli \
  --global \
  --yes
```

Para una instalacion local al proyecto, omite `--global` del mismo comando. Las instalaciones de proyecto son utiles cuando un equipo quiere registrar la version de la skill con el repositorio.

## Uso

Invoca la skill explicitamente:

```text
Use $better-fullstack to create a React + Vite app with Hono, SQLite, Drizzle, tRPC, Tailwind, DaisyUI, Pino, Vitest, and Biome.
```

O pregunta de forma natural. Los agentes que soportan matching implicito de skills pueden elegir la skill cuando la solicitud trate de scaffolding, planificacion o extension de un proyecto Better Fullstack.

## Qué hace la skill

1. Mapea el stack solicitado a partes de grafo y flags CLI de Better Fullstack.
2. Ejecuta un dry-run no interactivo de CLI.
3. Ejecuta el mismo comando sin `--dry-run` cuando el plan tiene éxito.
4. Usa `--no-install` y `--no-git` salvo que pidas explicitamente esos efectos secundarios.
5. Reporta el comando usado, cualquier ajuste de compatibilidad y los proximos comandos de install/test/run.

## Skill vs MCP

| Ruta | Mejor para |
| --- | --- |
| Agent skill | Scaffolding rapido guiado por CLI cuando el stack puede expresarse como flags o partes de grafo. |
| Servidor MCP | Busqueda estructurada de schema, comprobaciones de compatibilidad, planificacion dry-run y mutacion de proyectos existentes mediante tool calls. |

Ambas rutas usan el mismo generador Better Fullstack. La skill es la ruta más ligera; MCP es la ruta más estructurada.
