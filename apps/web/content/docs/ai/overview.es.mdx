---
title: Agentes de IA
description: Cómo los agentes de código con IA pueden usar Better Fullstack de forma segura y fiable.
---

Better Fullstack soporta tres flujos de trabajo con IA:

- Generar archivos de instrucciones locales al proyecto con `--ai-docs`.
- Instalar la skill de agente Better Fullstack para que los agentes de código usen la CLI en lugar de escribir archivos iniciales a mano.
- Permitir que un agente de código con IA use el servidor MCP de Better Fullstack para consultar el schema, comprobar compatibilidad, hacer dry-runs y crear proyectos.

## Flujo recomendado

1. Elige el ecosistema objetivo: `typescript`, `react-native`, `rust`, `python`, `go`, `java` o `elixir`.
2. Usa la skill de agente Better Fullstack, el servidor MCP o una referencia CLI explícita.
3. Valida compatibilidad antes de crear el scaffold.
4. Haz un dry-run para inspeccionar el árbol de archivos.
5. Genera el proyecto con la instalación de dependencias deshabilitada cuando un agente conduzca el flujo.
6. Da al usuario los comandos exactos de instalación, prueba y ejecución.

## Docs de IA generados

Usa `--ai-docs` para generar archivos de contexto locales al proyecto para agentes de código:

| Valor | Archivo generado |
| --- | --- |
| `claude-md` | `CLAUDE.md` |
| `agents-md` | `Agents.md` |
| `cursorrules` | `.cursorrules` |
| `none` | Sin docs de IA |

Se aceptan múltiples valores:

```npm
npm create better-fullstack@latest my-app -- --ai-docs claude-md agents-md
```

Usa `--ai-docs none` para deshabilitar estos archivos.

## Flujo con CLI

Para agentes sin acceso MCP, pasa flags explícitos y evita efectos secundarios:

```npm
npm create better-fullstack@latest my-app -- \
  --ecosystem typescript \
  --frontend next \
  --backend self \
  --runtime none \
  --database postgres \
  --orm drizzle \
  --auth better-auth \
  --api trpc \
  --db-setup none \
  --ai-docs agents-md \
  --version-channel stable \
  --no-install \
  --no-git
```

<Callout kind="info" title="Reglas CLI para agentes">
`--yes` acepta defaults y es incompatible con flags del stack core. Los flags explícitos saltan solo las categorías que cubren. Pasa `none` para categorías opcionales que quieras deshabilitar.
</Callout>

## Flujo con agent skill

La skill de agente Better Fullstack vive en este repositorio y enseña a los agentes soportados a usar la CLI con el flujo rápido medido: mapear el stack, ejecutar un dry-run, generar el proyecto sin efectos secundarios de instalación ni Git, y luego reportar comandos de seguimiento.

Consulta la [guía de setup de agent skill](/docs/ai/skills/) para comandos de instalación en Codex, Claude Code, Cursor, GitHub Copilot y Gemini CLI.

## Opciones de providers de IA

Los proyectos TypeScript pueden seleccionar una opción de IA con `--ai`: `vercel-ai`, `mastra`, `voltagent`, `langgraph`, `openai-agents`, `google-adk`, `modelfusion`, `langchain`, `llamaindex`, `tanstack-ai`, `ai-cli` o `none`.

Los proyectos Python usan la opción separada multi-select `--python-ai` con `langchain`, `llamaindex`, `openai-sdk`, `anthropic-sdk`, `langgraph`, `crewai` o `none`.

La ubicación de dependencias depende del framework y backend seleccionados. Los paquetes server-side se agregan al paquete servidor o a la app self-backend; las integraciones frontend soportadas reciben sus hooks o paquetes cliente correspondientes.

## Flujo MCP

El servidor MCP de Better Fullstack expone búsqueda de schema, comprobaciones de compatibilidad, previews dry-run, creación de proyectos y agregado de features como herramientas estructuradas. Consulta la [guía de setup MCP](/docs/ai/mcp/).

Esto es distinto del addon `mcp` del proyecto generado. El servidor ayuda a un agente a crear proyectos Better Fullstack; el addon escribe configuración MCP dentro de un proyecto generado.
