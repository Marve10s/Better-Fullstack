title: AI 代理
description: AI coding agents 如何安全可靠地使用 Better Fullstack。
---

Better Fullstack 支持三种 AI workflow：

- 使用 `--ai-docs` 生成项目本地指令文件。
- 安装 Better Fullstack agent skill，让 coding agents 使用 CLI，而不是手写 starter files。
- 让 AI coding agent 使用 Better Fullstack MCP server 进行 schema lookup、compatibility checks、dry-runs 和 project creation。

## 推荐 workflow

1. 选择目标生态系统：`typescript`、`react-native`、`rust`、`python`、`go`、`java` 或 `elixir`。
2. 使用 Better Fullstack agent skill、MCP server 或明确的 CLI reference。
3. 在 scaffold 前验证兼容性。
4. dry-run 以检查文件树。
5. 当 agent 驱动流程时，在禁用依赖安装的情况下 scaffold。
6. 告诉用户精确的 install、test 和 run commands。

## 生成的 AI docs

使用 `--ai-docs` 为 coding agents 生成项目本地上下文文件：

| 值 | 生成文件 |
| --- | --- |
| `claude-md` | `CLAUDE.md` |
| `agents-md` | `Agents.md` |
| `cursorrules` | `.cursorrules` |
| `none` | 不生成 AI docs |

接受多个值：

```npm
npm create better-fullstack@latest my-app -- --ai-docs claude-md agents-md
```

使用 `--ai-docs none` 禁用这些文件。

## CLI workflow

对于没有 MCP access 的 agents，传入明确 flags 并避免副作用：

```npm
npm create better-fullstack@latest my-app -- \
  --ecosystem typescript \
  --frontend next \
  --backend self \
  --runtime none \
  --database postgres \
  --orm drizzle \
  --auth better-auth \
  --api trpc \
  --db-setup none \
  --ai-docs agents-md \
  --version-channel stable \
  --no-install \
  --no-git
```

<Callout kind="info" title="面向 agents 的 CLI 规则">
`--yes` 会接受默认值，并且与 core stack flags 不兼容。明确 flags 只会跳过它们覆盖的分类。对你想禁用的可选分类传入 `none`。
</Callout>

## Agent skill workflow

Better Fullstack agent skill 位于此仓库中，并教受支持的 agents 使用 CLI 和经过基准验证的快速 workflow：映射 stack，运行 dry-run，在没有 install 或 Git 副作用的情况下 scaffold，然后报告后续命令。

请参阅 [agent skill setup guide](/docs/ai/skills/)，了解 Codex、Claude Code、Cursor、GitHub Copilot 和 Gemini CLI 的安装命令。

## AI provider 选项

TypeScript 项目可以通过 `--ai` 选择一个 AI 选项：`vercel-ai`、`mastra`、`voltagent`、`langgraph`、`openai-agents`、`google-adk`、`modelfusion`、`langchain`、`llamaindex`、`tanstack-ai`、`ai-cli` 或 `none`。

Python 项目使用单独的多选 `--python-ai` 选项，包含 `langchain`、`llamaindex`、`openai-sdk`、`anthropic-sdk`、`langgraph`、`crewai` 或 `none`。

依赖放置取决于选中的 framework 和 backend。Server-side packages 会添加到 server package 或 self-backend app；受支持的 frontend integrations 会获得对应的 client hooks 或 packages。

## MCP workflow

Better Fullstack MCP server 以结构化工具的形式暴露 schema lookup、compatibility checks、dry-run previews、project creation 和 feature additions。请参阅 [MCP setup guide](/docs/ai/mcp/)。

这不同于生成项目的 `mcp` addon。server 帮助 agent 创建 Better Fullstack 项目；addon 会在生成项目中写入 MCP 配置。
