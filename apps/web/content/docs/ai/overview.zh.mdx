title: AI 代理
description: AI 代码代理如何安全可靠地使用 Better Fullstack。
---

Better Fullstack 支持三种面向 AI 代理的工作流：

- 使用 `--ai-docs` 生成项目本地指令文件。
- 安装 Better Fullstack agent skill，让代码代理使用 CLI，而不是手写起步文件。
- 让 AI 代码代理使用 Better Fullstack MCP server 完成 schema 查询、兼容性检查、dry-run 预览和项目创建。

## 推荐工作流

1. 选择目标生态系统：`typescript`、`react-native`、`rust`、`python`、`go`、`java` 或 `elixir`。
2. 使用 Better Fullstack agent skill、MCP server，或一组明确的 CLI 参数。
3. 在生成项目之前验证兼容性。
4. 通过 dry-run 检查文件树。
5. 当流程由代理驱动时，在禁用依赖安装的情况下生成项目。
6. 告诉用户精确的安装、测试和运行命令。

## 生成的 AI docs

使用 `--ai-docs` 为代码代理生成项目本地上下文文件：

| 值 | 生成文件 |
| --- | --- |
| `claude-md` | `CLAUDE.md` |
| `agents-md` | `Agents.md` |
| `cursorrules` | `.cursorrules` |
| `none` | 不生成 AI docs |

接受多个值：

```npm
npm create better-fullstack@latest my-app -- --ai-docs claude-md agents-md
```

使用 `--ai-docs none` 禁用这些文件。

## CLI 工作流

对于没有 MCP 访问权限的代理，传入明确参数并避免副作用：

```npm
npm create better-fullstack@latest my-app -- \
  --ecosystem typescript \
  --frontend next \
  --backend self \
  --runtime none \
  --database postgres \
  --orm drizzle \
  --auth better-auth \
  --api trpc \
  --db-setup none \
  --ai-docs agents-md \
  --version-channel stable \
  --no-install \
  --no-git
```

<Callout kind="info" title="面向代理的 CLI 规则">
`--yes` 会接受默认值，并且与核心 stack 参数不兼容。明确参数只会跳过它们覆盖的分类。对你想禁用的可选分类传入 `none`。
</Callout>

## Agent skill 工作流

Better Fullstack agent skill 位于此仓库中，会教受支持的代理使用 CLI 和经过基准验证的快速流程：映射 stack、运行 dry-run、在没有安装或 Git 副作用的情况下生成项目，然后报告后续命令。

请参阅 [agent skill setup guide](/docs/ai/skills/)，了解 Codex、Claude Code、Cursor、GitHub Copilot 和 Gemini CLI 的安装命令。

## AI provider 选项

TypeScript 项目可以通过 `--ai` 选择一个 AI 选项：`vercel-ai`、`mastra`、`voltagent`、`langgraph`、`openai-agents`、`google-adk`、`modelfusion`、`langchain`、`llamaindex`、`tanstack-ai`、`ai-cli` 或 `none`。

Python 项目使用单独的多选 `--python-ai` 选项，包含 `langchain`、`llamaindex`、`openai-sdk`、`anthropic-sdk`、`langgraph`、`crewai` 或 `none`。

依赖放置取决于选中的框架和后端。Server-side packages 会添加到 server package 或 self-backend app；受支持的前端集成会获得对应的 client hooks 或 packages。

## MCP 工作流

Better Fullstack MCP server 以结构化工具的形式提供 schema 查询、兼容性检查、dry-run 预览、项目创建和功能添加。请参阅 [MCP setup guide](/docs/ai/mcp/)。

这不同于生成项目时选择的 `mcp` addon。server 帮助代理创建 Better Fullstack 项目；addon 则会在生成项目中写入 MCP 配置。
