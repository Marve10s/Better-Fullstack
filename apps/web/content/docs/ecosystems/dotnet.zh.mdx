title: .NET

---

.NET 项目会生成遵循标准 `dotnet` 约定的 ASP.NET Core 应用。为 Web 框架、数据访问、auth、API 风格、实时能力、后台任务、可观测性、缓存、测试和部署文件选择 .NET 选项。

## 前置要求

- .NET SDK 10 或更新版本。
- 使用带 `postgres` 数据库的 `ef-core` 时需要 PostgreSQL；SQLite 不需要服务器。
- 使用 `docker` 部署目标或 `testcontainers-dotnet` 时需要 Docker。
- 如果要初始化仓库，需要 Git。

## 脚本化示例

```npm
npm create better-fullstack@latest my-dotnet-app -- \
  --ecosystem dotnet \
  --dotnet-web-framework aspnet-minimal \
  --dotnet-orm ef-core \
  --dotnet-auth aspnet-identity \
  --dotnet-api minimal-api \
  --dotnet-testing xunit \
  --dotnet-job-queue hangfire \
  --dotnet-realtime signalr \
  --dotnet-observability serilog \
  --dotnet-caching redis \
  --dotnet-deploy docker \
  --database postgres \
  --ai-docs none \
  --no-install \
  --no-git
```

## 兼容性说明

- .NET 选项只在 `--ecosystem dotnet` 下生效。
- ASP.NET Core 是独立的 .NET Web 框架；TypeScript 前端和后端选项会被忽略。
- `--dotnet-testing` 和 `--dotnet-observability` 接受多个值。
- `ef-core` 从 `--database` 选择 provider：`postgres` 使用 Npgsql，其他任何值都回退到 SQLite。
- `aspnet-identity` 只有与 `ef-core` 搭配时才注册 Identity stores；没有它时，脚手架会注册普通的认证和授权服务。
- 设置 `--ecosystem dotnet` 时的非交互默认值：`aspnet-minimal`、`ef-core`、`aspnet-identity`、`minimal-api`、`xunit`、`signalr`、`serilog` 和 `docker`。

## 生成行为

- 项目包含 `.csproj`、`Program.cs`、`appsettings.json`、`.http` 请求文件、测试项目、`.gitignore` 和 README 中的 setup notes。
- EF Core 选择会把 `DbContext` 连接到 Npgsql 或 SQLite provider，并添加用于迁移的 design-time 包。
- 可选选择会添加可工作的 ASP.NET Core 代码，用于 Identity、Hot Chocolate GraphQL、gRPC 服务、SignalR hubs、Hangfire 或 Quartz.NET 任务、Serilog/NLog/OpenTelemetry pipelines、Redis 缓存和多阶段 Dockerfile。
