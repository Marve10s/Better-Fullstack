React Native 是专门的 mobile 生态。它生成 Expo 项目，并把 mobile choices 与 TypeScript web frontend、backend、database 和 deploy categories 分开。

## 前置条件

- Node.js 20+。
- npm、pnpm、Yarn 或 Bun。
- 当你想在 simulator 或 device 上验证时，需要 Expo-compatible iOS/Android tooling。

## 脚本化示例

```npm
npm create better-fullstack@latest mobile-app -- \
  --ecosystem react-native \
  --frontend native-bare \
  --mobile-navigation expo-router \
  --mobile-ui tamagui \
  --mobile-storage mmkv \
  --mobile-testing maestro-react-native-testing-library \
  --mobile-push expo-notifications \
  --mobile-ota expo-updates \
  --mobile-deep-linking expo-linking \
  --ai-docs none \
  --no-install
```

## Mobile scaffold categories

| Category | 值 |
| --- | --- |
| Native frontend | `native-bare` `native-uniwind` `native-unistyles` |
| Navigation | `expo-router` `react-navigation` `none` |
| UI | `tamagui` `gluestack-ui` `uniwind` `unistyles` `none` |
| Storage | `mmkv` `none` |
| Testing | `maestro` `react-native-testing-library` `maestro-react-native-testing-library` `none` |
| Push | `expo-notifications` `none` |
| OTA | `expo-updates` `none` |
| Deep linking | `expo-linking` `none` |

## 生成的集成

- 选择 deep linking 或 OTA 时生成带 scheme/runtime settings 的 Expo config。
- 根据 navigation 生成 Expo Router route files 或 React Navigation entrypoint。
- 为 Tamagui、Gluestack UI、Uniwind 或 Unistyles 生成 mobile UI provider/theme files。
- 选择 storage 时生成 MMKV client helpers。
- 生成 Jest Expo 和 React Native Testing Library 示例、Maestro flows，或两者都有。
- 为 Expo Notifications 生成 push notification registration helpers。
- 为 Expo Updates 生成 OTA update helpers。
- `.env.example` 使用 `EXPO_PUBLIC_` 约定存放 mobile client configuration。

## 兼容性说明

- React Native 不暴露 TypeScript web frontend、backend、database、web UI、web deploy 或 server deploy categories。
- Uniwind 和 Unistyles UI modes 与对应 Expo frontend variants 绑定。
- Tamagui 和 Gluestack UI 使用 bare Expo variant，以避免 styling setup 冲突。
- 当 mobile auth flow 需要 redirect URI 示例时，会自动启用 deep linking。
