---
title: Elixir
description: Phoenix 脚手架，包含 Ecto SQL、PostgreSQL 就绪配置、REST 或 Absinthe、实时能力、任务、测试和部署文件。
---

Elixir 项目会生成遵循 Mix 和 Phoenix 约定的 Phoenix 应用。为 LiveView、Ecto SQL、auth、API、实时能力、任务、测试、质量和部署文件选择 Elixir 选项。

## 前置要求

- 与 Phoenix 1.7 兼容的 Elixir 和 Erlang/OTP。
- 使用 `ecto-sql`、`phx-gen-auth` 或 `oban` 时需要 PostgreSQL。
- 如果要初始化仓库，需要 Git。

## 脚本化示例

```npm
npm create better-fullstack@latest my-phoenix-app -- \
  --ecosystem elixir \
  --elixir-web-framework phoenix-live-view \
  --elixir-orm ecto-sql \
  --elixir-auth phx-gen-auth \
  --elixir-api absinthe \
  --elixir-realtime presence \
  --elixir-jobs oban \
  --elixir-http req \
  --elixir-json jason \
  --elixir-observability telemetry \
  --elixir-testing ex_unit \
  --elixir-quality credo \
  --elixir-deploy docker \
  --ai-docs none \
  --no-install \
  --no-git
```

## 兼容性说明

- Elixir 选项只在 `--ecosystem elixir` 下生效。
- Phoenix 和 Phoenix LiveView 是独立的 Elixir Web 框架；TypeScript 前端和后端选项会被忽略。
- `phx-gen-auth` 和 Oban 需要 Ecto SQL，因为生成代码需要 Repo 和迁移。
- Oban 面向由 PostgreSQL 支撑的 Ecto SQL 项目。
- LiveView streams 需要 `phoenix-live-view`。
- 在匹配模板存在之前，图验证器会有意拒绝一些 schema 级选择：`ecto`、`ueberauth`、`guardian`、`nimble-options`、`nebulex`、`opentelemetry`、`prom_ex`、`mox`、`bypass`、`wallaby`、`fly` 和 `gigalixir`。CLI 和 builder 会把它们显示为不支持的工具消息，而不是生成不完整的 Phoenix 代码。

## 生成行为

- 项目包含 `mix.exs`、Phoenix 配置、router、endpoint、controllers、tests、`.env.example` 和 setup notes。
- Ecto SQL 选择会包含 Repo、迁移、schema/context 代码和 PostgreSQL 配置。
- 可选选择会添加可工作的 Phoenix 文件，用于 LiveView、Channels/Presence、Oban 或 Quantum 任务、Absinthe schema/resolvers、Req/Finch HTTP 客户端、Swoosh mailer、Cachex、Dockerfile 和 release 配置。
