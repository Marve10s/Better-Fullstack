Los proyectos Elixir generan aplicaciones Phoenix que siguen convenciones de Mix y Phoenix. Elige opciones de Elixir para LiveView, Ecto SQL, auth, APIs, tiempo real, jobs, testing, calidad y archivos de despliegue.

## Requisitos previos

- Elixir y Erlang/OTP compatibles con Phoenix 1.7.
- PostgreSQL al usar `ecto-sql`, `phx-gen-auth` u `oban`.
- Git si quieres inicializar un repositorio.

## Ejemplo con script

```npm
npm create better-fullstack@latest my-phoenix-app -- \
  --ecosystem elixir \
  --elixir-web-framework phoenix-live-view \
  --elixir-orm ecto-sql \
  --elixir-auth phx-gen-auth \
  --elixir-api absinthe \
  --elixir-realtime presence \
  --elixir-jobs oban \
  --elixir-http req \
  --elixir-json jason \
  --elixir-observability telemetry \
  --elixir-testing ex_unit \
  --elixir-quality credo \
  --elixir-deploy docker \
  --ai-docs none \
  --no-install \
  --no-git
```

## Notas de compatibilidad

- Las opciones de Elixir solo aplican con `--ecosystem elixir`.
- Phoenix y Phoenix LiveView son frameworks web independientes de Elixir; las opciones de frontend y backend de TypeScript se ignoran.
- `phx-gen-auth` y Oban requieren Ecto SQL porque el código generado necesita un Repo y migraciones.
- Oban apunta a proyectos Ecto SQL respaldados por PostgreSQL.
- LiveView streams requiere `phoenix-live-view`.
- Algunas opciones de nivel de schema se rechazan intencionalmente por el validador de grafo hasta que existan plantillas equivalentes: `ecto`, `ueberauth`, `guardian`, `nimble-options`, `nebulex`, `opentelemetry`, `prom_ex`, `mox`, `bypass`, `wallaby`, `fly` y `gigalixir`. La CLI y el builder las muestran como mensajes de herramienta no soportada en lugar de generar código Phoenix parcial.

## Comportamiento generado

- El proyecto incluye `mix.exs`, configuración Phoenix, router, endpoint, controladores, pruebas, `.env.example` y notas de setup.
- Las selecciones de Ecto SQL incluyen un Repo, migraciones, código de schema/context y configuración de PostgreSQL.
- Las selecciones opcionales agregan archivos Phoenix funcionales para LiveView, Channels/Presence, jobs Oban o Quantum, schema/resolvers de Absinthe, clientes HTTP Req/Finch, mailer Swoosh, Cachex, Dockerfile y configuración de release.
