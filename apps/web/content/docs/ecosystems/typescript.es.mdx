TypeScript es el ecosistema más amplio de Better Fullstack. Úsalo para apps web, APIs, workers, shells desktop, extensiones de navegador, sitios de documentación, integraciones de servicios y la mayoria de workflows de addons.

Los scaffolds mobile React Native/Expo viven en el ecosistema dedicado [React Native](/docs/ecosystems/react-native/).

## Prerrequisitos

- Node.js 20+.
- npm, pnpm, Yarn o Bun.
- Git si quieres inicializacion de repositorio.

## Ejemplo scriptado

```npm
npm create better-fullstack@latest my-app -- \
  --ecosystem typescript \
  --frontend tanstack-router \
  --backend hono \
  --runtime node \
  --database sqlite \
  --orm drizzle \
  --db-setup none \
  --api orpc \
  --auth better-auth \
  --addons turborepo \
  --examples none \
  --ai-docs none \
  --version-channel stable \
  --no-install
```

## Notas de compatibilidad

- tRPC es para frontends basados en React.
- `self` backend es para frameworks fullstack y se combina con `runtime none`.
- Qwik, Angular, Redwood y Fresh tienen soporte API/backend más estrecho que la tabla completa.
- El soporte de Workers runtime depende del backend.
- Las UI libraries suelen depender de la familia de frontend y del CSS framework. Por ejemplo, shadcn/ui está orientado a React y basado en Tailwind.
- Algunos providers tienen restricciones extra, como Polar requiriendo Better Auth y un web frontend.
- Algunas opciones se muestran por compatibilidad o soporte de dependencias antes de que cada ruta de template tenga la misma profundidad. Ejecuta el compatibility checker o Stack Builder antes de generar combinaciones grandes.
