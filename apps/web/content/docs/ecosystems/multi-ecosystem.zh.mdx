---
title: Multi-Ecosystem
description: 用跨生态系统的前端、后端、数据库和移动端部分组合一个 Better Fullstack 项目。
---

Multi-Ecosystem 模式把项目建模为相互连接的 stack parts，而不是一个扁平的生态系统选择。当产品跨语言时使用它，例如 TypeScript 前端、Go API、共享 PostgreSQL 数据库和 React Native 移动应用。

可以在 [Stack Builder](https://better-fullstack.dev/new?mode=multi) 中打开，或向 CLI 传入重复的 `--part` flags。

## Stack parts

每个 part 使用这种形状：

```txt
role:ecosystem:tool
owner.role:ecosystem:tool
```

顶层 roles 描述生成的应用或共享基础设施。带 owner 的 roles 会把能力挂到父级 part 上。

| 示例 | 含义 |
| --- | --- |
| `frontend:typescript:next` | 生成 TypeScript Next.js 前端。 |
| `backend:go:gin` | 生成 Go Gin 后端。 |
| `backend.orm:go:gorm` | 把 GORM 附加到选中的后端。 |
| `backend.api:rust:tonic` | 把 Tonic gRPC 附加到 Rust 后端。 |
| `database:universal:postgres` | 添加共享 PostgreSQL 数据库 part。 |
| `mobile:react-native:native-bare` | 生成 React Native 移动应用。 |

## 脚本化示例

```npm
npm create better-fullstack@latest my-mixed-app -- \
  --part frontend:typescript:next \
  --part backend:go:gin \
  --part backend.orm:go:gorm \
  --part backend.auth:go:jwt \
  --part database:universal:postgres \
  --part mobile:react-native:native-bare \
  --addons turborepo docker-compose
```

## 真相来源

Multi-ecosystem 项目会把 `stackParts` 写入 `bts.jsonc`。这些 parts 是生成结构、历史记录和可复现命令的真相来源。`backend`、`orm` 或 `frontend` 等旧字段仍可能作为从图派生的兼容性 fallback 出现。

生成的命令会把图保留为重复的 `--part` flags，因此可以在不依赖隐藏 builder 状态的情况下重建项目。

## Builder 流程

Stack Builder 的 Multi-Ecosystem 模式会依次走过：

1. Frontend - TypeScript Web 前端。
2. Backend - TypeScript、Rust、Python、Go、Java、Elixir 或 .NET 后端，以及带 owner 的后端能力。
3. Database - 共享的通用数据库选择。
4. Mobile - React Native 应用选择。
5. Finalize - addons、AI docs、install、Git 和 version 设置。

## 兼容性

- 当前图模型每个 role 只支持一个被选中的 primary part。
- 带 owner 的 capability parts 必须引用 owner，例如 `backend.orm:go:gorm`。
- capability 的生态系统通常需要与 owner 生态系统匹配。
- 通用 database parts 可以跨生态系统共享。
- 一些工具会提供隐含 parts。例如，被选中的后端可能提供数据库或路由假设，这些会出现在生成的 `stackParts` 中。
- compatibility checker 会在生成前验证图 parts，并报告 owner、重复和不支持工具的问题。
