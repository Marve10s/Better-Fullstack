Los proyectos .NET generan aplicaciones ASP.NET Core que siguen convenciones estandar de `dotnet`. Elige opciones de .NET para el framework web, acceso a datos, auth, estilo de API, tiempo real, jobs en background, observabilidad, cache, testing y archivos de despliegue.

## Requisitos previos

- .NET SDK 10 o superior.
- PostgreSQL al usar `ef-core` con la base de datos `postgres`; SQLite no necesita servidor.
- Docker al usar el destino de despliegue `docker` o `testcontainers-dotnet`.
- Git si quieres inicializar un repositorio.

## Ejemplo con script

```npm
npm create better-fullstack@latest my-dotnet-app -- \
  --ecosystem dotnet \
  --dotnet-web-framework aspnet-minimal \
  --dotnet-orm ef-core \
  --dotnet-auth aspnet-identity \
  --dotnet-api minimal-api \
  --dotnet-testing xunit \
  --dotnet-job-queue hangfire \
  --dotnet-realtime signalr \
  --dotnet-observability serilog \
  --dotnet-caching redis \
  --dotnet-deploy docker \
  --database postgres \
  --ai-docs none \
  --no-install \
  --no-git
```

## Notas de compatibilidad

- Las opciones de .NET solo aplican con `--ecosystem dotnet`.
- ASP.NET Core es un framework web .NET independiente; las opciones de frontend y backend de TypeScript se ignoran.
- `--dotnet-testing` y `--dotnet-observability` aceptan múltiples valores.
- `ef-core` selecciona el provider desde `--database`: `postgres` usa Npgsql, cualquier otro valor cae a SQLite.
- `aspnet-identity` registra stores de Identity solo cuando se combina con `ef-core`; sin el, el scaffold registra servicios simples de autenticación y autorizacion.
- Defaults no interactivos cuando se configura `--ecosystem dotnet`: `aspnet-minimal`, `ef-core`, `aspnet-identity`, `minimal-api`, `xunit`, `signalr`, `serilog` y `docker`.

## Comportamiento generado

- El proyecto incluye un `.csproj`, `Program.cs`, `appsettings.json`, un archivo de requests `.http`, un proyecto de tests, `.gitignore` y notas de setup en el README.
- Las selecciones de EF Core conectan un `DbContext` con el provider Npgsql o SQLite más el paquete design-time para migraciones.
- Las selecciones opcionales agregan código ASP.NET Core funcional para Identity, GraphQL con Hot Chocolate, servicios gRPC, hubs SignalR, jobs Hangfire o Quartz.NET, pipelines Serilog/NLog/OpenTelemetry, cache Redis y un Dockerfile multi-stage.
