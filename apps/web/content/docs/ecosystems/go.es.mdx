Los proyectos Go usan Go modules y plantillas nativas de Go. Elige opciones de Go para servicios HTTP, gRPC, CLIs, logging, ORMs y helpers de autenticación.

## Requisitos previos

- Toolchain de Go.
- Git si quieres inicializar un repositorio.

## Ejemplo con script

```npm
npm create better-fullstack@latest my-go-api -- \
  --ecosystem go \
  --go-web-framework gin \
  --go-orm gorm \
  --go-logging zap \
  --go-auth jwt \
  --go-api none \
  --go-cli none \
  --go-testing testify \
  --go-realtime gorilla-websocket \
  --go-message-queue nats \
  --go-caching redis \
  --go-config viper \
  --go-observability opentelemetry \
  --auth none \
  --ai-docs none \
  --version-channel stable \
  --no-install \
  --no-git
```

## Categorias de Go

| Categoria | Valores |
| --- | --- |
| Framework web | `gin` `echo` `fiber` `chi` `none` |
| ORM | `gorm` `sqlc` `ent` `none` |
| API | `grpc-go` `gqlgen` `none` |
| CLI | `cobra` `bubbletea` `urfave-cli` `none` |
| Logging | `zap` `zerolog` `slog` `logrus` `none` |
| Helpers de auth para Go | `casbin` `jwt` `goth` `none` |
| Testing (seleccion multiple) | `testify` `gomock` `none` |
| Tiempo real | `gorilla-websocket` `centrifuge` `none` |
| Cola de mensajes | `nats` `watermill` `none` |
| Cache | `redis` `ristretto` `none` |
| Configuracion | `viper` `koanf` `none` |
| Observabilidad | `opentelemetry` `none` |

## Separacion de auth

Go tiene dos campos relacionados con autenticación:

- `--go-auth casbin|jwt|none` agrega helpers de autenticación nativos de Go.
- `--auth go-better-auth` selecciona GoBetterAuth mediante el campo global de auth.

Otros proveedores globales de auth como Clerk, NextAuth, Stack Auth, Supabase Auth y Auth0 estan orientados a TypeScript y no deben presentarse como soporte general para scaffolds de Go.

## Comportamiento generado

- `go mod tidy` es la ruta de instalacion/verificacion cuando la instalacion está habilitada.
- Los archivos de gRPC, Cobra y Bubble Tea se incluyen solo cuando se seleccionan.
- Las selecciones de testing, tiempo real, cola de mensajes, cache, configuración y observabilidad generan paquetes independientes bajo `internal/` (más pruebas de ejemplo para `testify`/`gomock`).
- `gqlgen` genera `graph/schema.graphqls` + `gqlgen.yml`; ejecuta `go generate ./graph` para producir el código del servidor GraphQL (igual que el flujo de generacion de código de Ent).
- `goth` registra proveedores OAuth desde credenciales de entorno con formato `GOOGLE_CLIENT_ID`/`GITHUB_CLIENT_ID`.
- Los archivos de instrucciones para IA siguen disponibles mediante `--ai-docs`, incluido `--ai-docs none`.
