TypeScript 是 Better Fullstack 最广的生态。可用于 web apps、APIs、workers、desktop shells、浏览器扩展、文档站点、service integrations，以及大多数 addon workflows。

React Native/Expo mobile scaffolds 位于专门的 [React Native](/docs/ecosystems/react-native/) 生态。

## 前置条件

- Node.js 20+。
- npm、pnpm、Yarn 或 Bun。
- 如果想初始化仓库，需要 Git。

## 脚本化示例

```npm
npm create better-fullstack@latest my-app -- \
  --ecosystem typescript \
  --frontend tanstack-router \
  --backend hono \
  --runtime node \
  --database sqlite \
  --orm drizzle \
  --db-setup none \
  --api orpc \
  --auth better-auth \
  --addons turborepo \
  --examples none \
  --ai-docs none \
  --version-channel stable \
  --no-install
```

## 兼容性说明

- tRPC 面向基于 React 的 frontends。
- `self` backend 用于 fullstack frameworks，并与 `runtime none` 搭配。
- Qwik、Angular、Redwood 和 Fresh 的 API/backend 支持比完整选项表更窄。
- Workers runtime 支持取决于 backend。
- UI libraries 通常依赖 frontend family 和 CSS framework。例如 shadcn/ui 面向 React，并基于 Tailwind。
- 一些 providers 有额外约束，例如 Polar 需要 Better Auth 和 web frontend。
- 有些选项会先作为兼容或依赖支持暴露，然后每条 template path 再逐步补齐深度。生成大型组合前先运行 compatibility checker 或 Stack Builder。
