Los proyectos Python usan `uv` y plantillas nativas de Python. Elige opciones Python para frameworks API, ORMs, validación, librerias de IA, helpers de auth, task queues, GraphQL y tooling de calidad.

## Prerrequisitos

- Python.
- `uv`.
- Git si quieres inicializacion de repositorio.

## Ejemplo scriptado

```npm
npm create better-fullstack@latest my-python-api -- \
  --ecosystem python \
  --python-web-framework fastapi \
  --python-orm sqlalchemy \
  --python-validation pydantic \
  --python-ai none \
  --python-auth jwt \
  --python-task-queue none \
  --python-graphql none \
  --python-quality ruff \
  --python-testing pytest hypothesis \
  --python-caching redis \
  --python-realtime python-socketio \
  --python-observability opentelemetry \
  --python-cli typer rich \
  --ai-docs none \
  --version-channel stable \
  --no-install \
  --no-git
```

## Comportamiento generado

- `uv sync` es la ruta de instalacion cuando install está activado.
- `pythonAi` es multi-select y está separada de `--ai` de TypeScript.
- Los archivos Python condicionales vacios se omiten cuando las opciones seleccionadas no los necesitan.
- Los archivos de instrucciones para IA siguen disponibles mediante `--ai-docs`, incluido `--ai-docs none`.
