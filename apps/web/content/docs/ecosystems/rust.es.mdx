Los proyectos Rust usan Cargo y plantillas nativas del lenguaje. Elige opciones Rust para servicios backend, CLIs, GraphQL/gRPC, frontends WebAssembly, logging, caching y helpers de auth.

## Prerrequisitos

- Rust toolchain y Cargo.
- Git si quieres inicializacion de repositorio.

## Ejemplo scriptado

```npm
npm create better-fullstack@latest my-rust-api -- \
  --ecosystem rust \
  --rust-web-framework axum \
  --rust-orm sqlx \
  --rust-libraries serde validator \
  --rust-logging tracing \
  --rust-error-handling anyhow-thiserror \
  --rust-api none \
  --rust-frontend none \
  --rust-cli none \
  --rust-caching none \
  --rust-auth none \
  --rust-realtime tokio-tungstenite \
  --rust-message-queue lapin \
  --rust-observability opentelemetry \
  --rust-templating askama \
  --ai-docs none \
  --version-channel stable \
  --no-install \
  --no-git
```

## Comportamiento generado

- `cargo build` es la ruta de instalacion/verificacion cuando install está activado.
- Los archivos de template para Leptos, Dioxus, Tonic, Clap y Ratatui se incluyen solo cuando se seleccionan sus opciones.
- `rustLibraries` es multi-select; pasa `none` u omite las librerias que no quieras.
- Los archivos de instrucciones para IA siguen disponibles mediante `--ai-docs`, incluido `--ai-docs none`.
