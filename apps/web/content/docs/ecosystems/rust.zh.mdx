Rust 项目使用 Cargo 和语言原生 templates。为 backend services、CLIs、GraphQL/gRPC、WebAssembly frontends、logging、caching 和 auth helpers 选择 Rust options。

## 前置条件

- Rust toolchain 和 Cargo。
- 如果想初始化仓库，需要 Git。

## 脚本化示例

```npm
npm create better-fullstack@latest my-rust-api -- \
  --ecosystem rust \
  --rust-web-framework axum \
  --rust-orm sqlx \
  --rust-libraries serde validator \
  --rust-logging tracing \
  --rust-error-handling anyhow-thiserror \
  --rust-api none \
  --rust-frontend none \
  --rust-cli none \
  --rust-caching none \
  --rust-auth none \
  --rust-realtime tokio-tungstenite \
  --rust-message-queue lapin \
  --rust-observability opentelemetry \
  --rust-templating askama \
  --ai-docs none \
  --version-channel stable \
  --no-install \
  --no-git
```

## 生成行为

- 启用 install 时，`cargo build` 是安装/验证路径。
- Leptos、Dioxus、Tonic、Clap 和 Ratatui template files 只在选择对应选项时包含。
- `rustLibraries` 是 multi-select；传 `none` 或省略不想要的 libraries。
- AI instruction files 仍可通过 `--ai-docs` 使用，包括 `--ai-docs none`。
