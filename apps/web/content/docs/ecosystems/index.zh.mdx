title: 生态系统
description: 支持的语言生态，以及 Better Fullstack 为每个生态生成什么。
---

Better Fullstack 用一个 CLI 覆盖多个生态。每个生态都有自己的 scaffold surface、toolchain 和兼容性规则。

当产品 stack 跨语言时，使用 [Multi-Ecosystem mode](/docs/ecosystems/multi-ecosystem/)。它把项目建模成 stack parts，例如 `frontend:typescript:next`、`backend:go:gin`、`backend.orm:go:gorm` 和 `database:universal:postgres`。

| 生态系统 | 最适合 | 说明 |
| --- | --- | --- |
| [TypeScript](/docs/ecosystems/typescript/) | Fullstack web、APIs、workers、desktop，以及最大的 web integration matrix。 | Web 选项是 TypeScript-first。 |
| [React Native](/docs/ecosystems/react-native/) | 带 navigation、UI、storage、testing、push、OTA 和 deep linking 的 Expo mobile apps。 | 使用 React Native 生态 surface，而不是 TypeScript web frontend categories。 |
| [Rust](/docs/ecosystems/rust/) | Backend services、CLI apps、GraphQL/gRPC、Rust web frontends 和强类型 libraries。 | 使用 Cargo；templates 会根据所选 Rust options 条件生成。 |
| [Python](/docs/ecosystems/python/) | API 和 AI-oriented services，包含 framework、ORM、validation、queue、GraphQL 和 quality choices。 | 使用 `uv`；Python AI 是独立的 multi-select category。 |
| [Go](/docs/ecosystems/go/) | API services、gRPC、CLIs、logging、ORM 和 Go auth helpers。 | 使用 Go modules；GoBetterAuth 通过全局 `--auth go-better-auth` 选择。 |
| [Java](/docs/ecosystems/java/) | Spring Boot、Quarkus 或 plain Java 项目，包含 Maven/Gradle、persistence、auth、libraries 和 tests。 | 选择 build tool 时 Java 使用 Maven 或 Gradle wrappers。 |
| [Elixir](/docs/ecosystems/elixir/) | Phoenix 和 Phoenix LiveView apps，包含 Ecto、Absinthe、realtime、jobs、testing、quality 和 deploy choices。 | 使用 Mix；少数 schema-level options 会在 templates 跟上前保持禁用。 |
| [.NET](/docs/ecosystems/dotnet/) | ASP.NET Core services，包含 EF Core 或 Dapper、Identity、Minimal APIs 或 Hot Chocolate、SignalR、jobs 和 tests。 | 使用 .NET SDK；testing 和 observability 是 multi-select categories。 |

## 默认 CLI baseline

接受 defaults 会创建一个 TypeScript web 项目。package manager 会跟随启动 CLI 时使用的 package manager。

```json
{
  "ecosystem": "typescript",
  "frontend": ["tanstack-router"],
  "backend": "hono",
  "runtime": "bun",
  "database": "sqlite",
  "orm": "drizzle",
  "auth": "better-auth",
  "api": "trpc",
  "cssFramework": "tailwind",
  "uiLibrary": "shadcn-ui",
  "testing": "vitest",
  "addons": ["turborepo"],
  "versionChannel": "stable",
  "aiDocs": ["claude-md"]
}
```

Runtime 和 package manager 是独立选择。例如，生成的 TypeScript 项目可以用 npm、pnpm、Yarn 或 Bun 安装依赖，同时选择 Bun 作为 application runtime。
