The `create` command scaffolds a new project from your selected stack. You can run the interactive wizard or pass all flags explicitly to skip every prompt.

<PMTabs
  npm="npm create better-fullstack@latest [project-name] -- [flags]"
  pnpm="pnpm create better-fullstack@latest [project-name] [flags]"
  bun="bun create better-fullstack@latest [project-name] [flags]"
  yarn="yarn create better-fullstack@latest [project-name] [flags]"
/>

<Callout kind="info" title="npm flag forwarding">
With `npm create`, pass all Better Fullstack flags after a `--` separator. The other launchers forward flags directly.
</Callout>

## Compatibility rules

<Callout kind="warning" title="Know before you scaffold">
A few combinations are hard-incompatible:

- **tRPC** (`--api trpc`) only works with React-based frontends: `tanstack-router`, `react-router`, `react-vite`, `tanstack-start`, `next`.
- **Angular** has its own HttpClient — use `--api none`.
- **Qwik** has built-in server actions — use `--backend none --api none`.
- **`--backend self`** only works with `next`, `tanstack-start`, `astro`, `nuxt`, `svelte`, `solid-start`. Requires `--runtime none`.
- **NestJS / AdonisJS** require `--runtime node`.
- **`--yes`** (accept defaults) is incompatible with explicit stack flags — omit it when you pass `--frontend`, `--backend`, or similar.
- **`--ai-docs none`** hangs. Use `--ai-docs claude-md` or omit the flag entirely.
</Callout>

## Global flags

These flags apply to all ecosystems.

| Flag | Description |
| --- | --- |
| `--ecosystem` | Language ecosystem: `typescript` `rust` `python` `go` `java`. Defaults to `typescript`. |
| `--template` | Use a CLI shorthand preset: `t3` `mern` `pern` `uniwind`. |
| `--addons` _(multi)_ | Extra tooling: `mcp` `skills` `turborepo` `starlight` `fumadocs` `biome` `oxlint` `ultracite` `lefthook` `husky` `pwa` `tauri` `wxt` `msw` `storybook` `tanstack-query` `tanstack-table` `tanstack-virtual` `tanstack-db` `tanstack-pacer` `opentui` `ruler` `none`. |
| `--examples` _(multi)_ | Starter examples to scaffold: `ai` `chat-sdk` `tanstack-showcase` `none`. |
| `--ai-docs` _(multi)_ | AI context files to generate: `claude-md` `agents-md` `cursorrules`. **Do not use `none` — it hangs. Omit the flag instead.** |
| `--package-manager` | `npm` `pnpm` `bun` `yarn`. |
| `--web-deploy` | Web deployment target (see TypeScript deploy table above). |
| `--server-deploy` | Server deployment target. |
| `--version-channel` | Dependency version pinning: `stable` `latest` `beta`. |
| `--dry-run` | Preview the generated file tree without writing to disk. |
| `--yes` | Accept all defaults. **Incompatible with explicit stack flags.** |
| `--no-install` | Skip dependency installation. Run the installer separately after reviewing generated files. |
| `--no-git` | Skip Git initialization. |
| `--disable-analytics` | Opt out of anonymous usage analytics. |
| `--verbose` | Print detailed scaffold result output. |

## Automation tip

<Callout kind="tip" title="Scripts and AI agents">
For CI pipelines and AI-driven scaffolding, always pass `--no-install` and run the package manager separately after reviewing the generated files. Supply every flag explicitly to avoid interactive prompts — see the per-ecosystem examples above. The compatibility checker runs before generation and will reject invalid combinations with a clear error.
</Callout>
