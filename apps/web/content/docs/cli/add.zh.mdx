---
title: 添加功能
description: 向现有 Better Fullstack 项目添加支持的 addons。
---

`add` 命令会更新现有 Better Fullstack 项目。它读取 `bts.jsonc`，验证请求的 addons，写入支持的文件，并更新项目配置。

```npm
npm create better-fullstack@latest add -- --addons mcp skills
```

## add 会做什么

- 从当前项目或 `--project-dir` 读取 `bts.jsonc`。
- 省略 `--addons` 时提示选择兼容 addons。
- 跳过已经安装的 addons。
- 选择新 addon 时写入 addon templates 和依赖变更。
- 跳过依赖安装时打印安装命令。

<Callout kind="warn" title="Addon-first 命令">
`add` 目前在没有选择新 addon 时会退出。尽可能在 `create` 阶段选择 deployment targets；仅 deployment 的 `add --web-deploy ...` 或 `add --server-deploy ...` flows 还不完整。
</Callout>

## Flags

| Flag | 值或行为 |
| --- | --- |
| `--addons` | `pwa` `tauri` `starlight` `biome` `lefthook` `husky` `ruler` `mcp` `skills` `turborepo` `fumadocs` `ultracite` `oxlint` `opentui` `wxt` `msw` `storybook` `swr` `tanstack-query` `tanstack-table` `tanstack-virtual` `tanstack-db` `tanstack-pacer` `backend-utils` `docker-compose` `none` |
| `--project-dir` | 生成项目的路径。默认是当前目录。 |
| `--install` | 添加文件后安装依赖。对 `add` 默认是 false。 |
| `--package-manager` | `npm` `pnpm` `bun` `yarn` |
| `--web-deploy` | addon 处理后记录 web deployment target。 |
| `--server-deploy` | addon 处理后记录 server deployment target。 |

## Addon 参考

| Addon | 用途 |
| --- | --- |
| `mcp` | 为代码代理生成项目内 MCP server 配置。 |
| `skills` | 生成精选 coding-agent skill 文件。 |
| `turborepo` | Monorepo 编排。 |
| `docker-compose` | 为支持的 stacks 提供本地服务编排。 |
| `biome` `oxlint` `ultracite` `lefthook` `husky` `ruler` | 代码质量和 hooks。 |
| `starlight` `fumadocs` | 与 stack 兼容时生成文档站点。 |
| `tauri` `pwa` `wxt` | Desktop、progressive web app 和浏览器扩展支持。 |
| `msw` `storybook` | Testing 和组件开发工具。 |
| `swr` `tanstack-query` `tanstack-table` `tanstack-virtual` `tanstack-db` `tanstack-pacer` | 数据获取和 UI 工具。 |
| `backend-utils` | 为 Hono、Express、Fastify、Elysia、feTS 或 NestJS servers 提供 asyncHandler、ApiResponse 和全局错误 handler。 |
| `opentui` | Terminal UI 支持。 |

## 幂等性

`bts.jsonc` 是已安装 addons 的 source of truth。如果请求的所有 addons 都已经存在，`add` 会报告没有选择新 addon，并且不会应用无关变更。

## 推荐 workflow

1. Commit 或 stash 当前工作。
2. 使用显式 addon flags 运行 `add`。
3. 审查生成的文件变更。
4. 如果命令跳过安装，则安装依赖。
5. 运行项目的 test、lint 或 typecheck 命令。
