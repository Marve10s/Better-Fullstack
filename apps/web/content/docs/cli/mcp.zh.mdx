`mcp` 命令通过 stdio 启动 Better Fullstack MCP server。代理可以用它检查 schemas、验证兼容性、预览生成文件、创建项目，并向现有项目添加支持的功能。

<PMTabs
  npm="npx -y create-better-fullstack@latest mcp"
  pnpm="pnpm dlx create-better-fullstack@latest mcp"
  bun="bunx create-better-fullstack@latest mcp"
  yarn="yarn dlx create-better-fullstack@latest mcp"
/>

服务器使用 stdio transport。配置你的 MCP client 启动该命令，并通过 standard input/output 通信。

## Client 示例

使用 npm 的 Claude Code：

```bash
claude mcp add --transport stdio better-fullstack -- npx -y create-better-fullstack@latest mcp
```

如果你偏好 pnpm、Bun 或 Yarn，把 `--` 之后的命令替换成匹配的 launcher：

| Package manager | Launcher |
| --- | --- |
| npm | `npx -y create-better-fullstack@latest mcp` |
| pnpm | `pnpm dlx create-better-fullstack@latest mcp` |
| Bun | `bunx create-better-fullstack@latest mcp` |
| Yarn | `yarn dlx create-better-fullstack@latest mcp` |

使用 npm 的通用 MCP client config：

```json
{
  "mcpServers": {
    "better-fullstack": {
      "command": "npx",
      "args": ["-y", "create-better-fullstack@latest", "mcp"]
    }
  }
}
```

其他 package managers 的等价 command/args：

| Package manager | `command` | `args` |
| --- | --- | --- |
| pnpm | `pnpm` | `["dlx", "create-better-fullstack@latest", "mcp"]` |
| Bun | `bunx` | `["create-better-fullstack@latest", "mcp"]` |
| Yarn | `yarn` | `["dlx", "create-better-fullstack@latest", "mcp"]` |

## 工具表面

| 工具 | 用途 |
| --- | --- |
| `bfs_get_guidance` | 返回 workflow 规则和字段语义。 |
| `bfs_get_schema` | 返回一个 category 或所有 categories 的有效选项。 |
| `bfs_check_compatibility` | 在生成前验证 stack selections。 |
| `bfs_plan_project` | 在内存中 dry-run 项目生成。 |
| `bfs_create_project` | 将新项目写入磁盘。 |
| `bfs_plan_addition` | 为现有项目规划 feature additions。 |
| `bfs_add_feature` | 向现有项目添加支持的功能。 |

项目规划、创建和添加响应会在 generated file 或 mutation summary 之外，包含 graph metadata（`graphSummary`、`effectiveStack` 和 `stackPartSpecs`）。

查看 [MCP tools reference](/docs/ai/mcp-tools/) 了解 inputs、outputs 和安全 workflows。

## 安全说明

- `bfs_plan_project` 不写入文件。
- `bfs_create_project` 写入文件，但不安装依赖。
- `bfs_add_feature` 会修改现有 Better Fullstack 项目，并且不安装依赖。
- Agents 应该在创建项目之前调用 `bfs_get_guidance` 和 `bfs_check_compatibility`。
