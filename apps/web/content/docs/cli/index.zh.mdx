Better Fullstack 的命令面很小。大多数用户只需要 `create`、`add` 和 `history`；AI-agent workflows 使用 `mcp`。

## 命令

| 命令 | 用途 |
| --- | --- |
| `create` | 从 stack configuration 生成新项目。这是默认命令。 |
| `add` | 向现有 Better Fullstack 项目添加支持的 addons 或 deployment settings。 |
| `history` | 显示、导出或清理本地 scaffold 历史和可复现命令。 |
| `mcp` | 通过 stdio 为 AI 代码代理启动 Better Fullstack MCP server。 |

## 工具命令

| 命令 | 用途 |
| --- | --- |
| `docs` | 在浏览器中打开公开的 Better Fullstack 文档站点。 |
| `builder` | 打开交互式 Stack Builder。 |
| `sponsors` | 显示 Better Fullstack sponsors。 |

## 运行工具命令

打开 docs：

<PMTabs
  npm="npx -y create-better-fullstack@latest docs"
  pnpm="pnpm dlx create-better-fullstack@latest docs"
  bun="bunx create-better-fullstack@latest docs"
  yarn="yarn dlx create-better-fullstack@latest docs"
/>

## Maintainer 命令

`update-deps` 用于 maintainers 检查和更新依赖版本。它有意不出现在普通用户 workflows 中。
