El comando `mcp` inicia el servidor MCP de Better Fullstack sobre stdio. Los agentes pueden usarlo para inspeccionar schemas, validar compatibilidad, previsualizar archivos generados, crear proyectos y agregar features soportadas a proyectos existentes.

<PMTabs
  npm="npx -y create-better-fullstack@latest mcp"
  pnpm="pnpm dlx create-better-fullstack@latest mcp"
  bun="bunx create-better-fullstack@latest mcp"
  yarn="yarn dlx create-better-fullstack@latest mcp"
/>

El servidor usa transporte stdio. Configura tu cliente MCP para lanzar el comando y comunicarse por standard input/output.

## Ejemplos de cliente

Claude Code con npm:

```bash
claude mcp add --transport stdio better-fullstack -- npx -y create-better-fullstack@latest mcp
```

Para pnpm, Bun o Yarn, reemplaza el comando después de `--` por el launcher correspondiente:

| Package manager | Launcher |
| --- | --- |
| npm | `npx -y create-better-fullstack@latest mcp` |
| pnpm | `pnpm dlx create-better-fullstack@latest mcp` |
| Bun | `bunx create-better-fullstack@latest mcp` |
| Yarn | `yarn dlx create-better-fullstack@latest mcp` |

Config generica de cliente MCP con npm:

```json
{
  "mcpServers": {
    "better-fullstack": {
      "command": "npx",
      "args": ["-y", "create-better-fullstack@latest", "mcp"]
    }
  }
}
```

Comando/args equivalentes para otros package managers:

| Package manager | `command` | `args` |
| --- | --- | --- |
| pnpm | `pnpm` | `["dlx", "create-better-fullstack@latest", "mcp"]` |
| Bun | `bunx` | `["create-better-fullstack@latest", "mcp"]` |
| Yarn | `yarn` | `["dlx", "create-better-fullstack@latest", "mcp"]` |

## Superficie de herramientas

| Herramienta | Propósito |
| --- | --- |
| `bfs_get_guidance` | Devuelve reglas de workflow y semantica de campos. |
| `bfs_get_schema` | Devuelve opciones validas para una categoría o todas las categorías. |
| `bfs_check_compatibility` | Valida selecciones de stack antes de generar. |
| `bfs_plan_project` | Hace dry-run de la generacion en memoria. |
| `bfs_create_project` | Escribe un proyecto nuevo en disco. |
| `bfs_plan_addition` | Planifica adiciones de features para un proyecto existente. |
| `bfs_add_feature` | Agrega features soportadas a un proyecto existente. |

Las respuestas de planificacion, creación y adición incluyen metadata de grafo (`graphSummary`, `effectiveStack` y `stackPartSpecs`) junto al resumen de archivos generados o mutaciones.

Consulta la [referencia de herramientas MCP](/docs/ai/mcp-tools/) para inputs, outputs y workflows seguros.

## Notas de seguridad

- `bfs_plan_project` no escribe archivos.
- `bfs_create_project` escribe archivos pero no instala dependencias.
- `bfs_add_feature` muta un proyecto Better Fullstack existente y no instala dependencias.
- Los agentes deberían llamar a `bfs_get_guidance` y `bfs_check_compatibility` antes de crear proyectos.

## Comandos relacionados

- `docs` abre el sitio público de documentación.
- `builder` abre el Stack Builder interactivo.
- `sponsors` muestra sponsors de Better Fullstack.

Consulta la [referencia CLI](/docs/cli/) para ejemplos de estos comandos utilitarios.
