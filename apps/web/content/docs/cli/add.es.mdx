---
title: Agregar features
description: Agrega addons soportados a proyectos Better Fullstack existentes.
---

El comando `add` actualiza un proyecto Better Fullstack existente. Lee `bts.jsonc`, valida los addons solicitados, escribe archivos soportados y actualiza la configuración del proyecto.

```npm
npm create better-fullstack@latest add -- --addons mcp skills
```

## Qué hace add

- Lee `bts.jsonc` desde el proyecto actual o desde `--project-dir`.
- Pregunta por addons compatibles cuando se omite `--addons`.
- Omite addons que ya estan instalados.
- Escribe templates de addon y cambios de dependencias cuando se selecciona un addon nuevo.
- Imprime un comando de instalacion cuando se salta la instalacion de dependencias.

<Callout kind="warn" title="Comando addon-first">
`add` actualmente sale cuando no se selecciona ningun addon nuevo. Elige targets de deployment durante `create` cuando sea posible; los flujos solo-deployment `add --web-deploy ...` o `add --server-deploy ...` aun no estan completos.
</Callout>

## Flags

| Flag | Valores o comportamiento |
| --- | --- |
| `--addons` | `pwa` `tauri` `starlight` `biome` `lefthook` `husky` `ruler` `mcp` `skills` `turborepo` `fumadocs` `ultracite` `oxlint` `opentui` `wxt` `msw` `storybook` `swr` `tanstack-query` `tanstack-table` `tanstack-virtual` `tanstack-db` `tanstack-pacer` `backend-utils` `docker-compose` `none` |
| `--project-dir` | Ruta al proyecto generado. Por defecto usa el directorio actual. |
| `--install` | Instala dependencias después de agregar archivos. Por defecto es false para `add`. |
| `--package-manager` | `npm` `pnpm` `bun` `yarn` |
| `--web-deploy` | Registra un target de deployment web después de procesar addons. |
| `--server-deploy` | Registra un target de deployment server después de procesar addons. |

## Referencia de addons

| Addon | Propósito |
| --- | --- |
| `mcp` | Configuracion MCP del proyecto generado para agentes de código. |
| `skills` | Archivos de skills curadas para agentes de código. |
| `turborepo` | Orquestacion monorepo. |
| `docker-compose` | Orquestacion de servicios locales para stacks soportados. |
| `biome` `oxlint` `ultracite` `lefthook` `husky` `ruler` | Calidad de código y hooks. |
| `starlight` `fumadocs` | Sitios de documentación cuando son compatibles con el stack. |
| `tauri` `pwa` `wxt` | Soporte desktop, progressive web app y extension de navegador. |
| `msw` `storybook` | Utilidades de testing y desarrollo de componentes. |
| `swr` `tanstack-query` `tanstack-table` `tanstack-virtual` `tanstack-db` `tanstack-pacer` | Utilidades de fetching de datos y UI. |
| `backend-utils` | asyncHandler, ApiResponse y handler global de errores para servidores Hono, Express, Fastify, Elysia, feTS o NestJS. |
| `opentui` | Soporte de terminal UI. |

## Idempotencia

`bts.jsonc` es la fuente de verdad para addons instalados. Si todos los addons solicitados ya estan presentes, `add` informa que no se seleccionaron addons nuevos y sale sin aplicar cambios no relacionados.

## Workflow recomendado

1. Haz commit o stash del trabajo actual.
2. Ejecuta `add` con flags explicitos de addon.
3. Revisa los cambios de archivos generados.
4. Instala dependencias si el comando salto la instalacion.
5. Ejecuta el comando de test, lint o typecheck del proyecto.
