Better Fullstack tiene una superficie de comandos pequena. La mayoria de usuarios solo necesita `create`, `add` e `history`; los workflows de agentes de IA usan `mcp`.

## Comandos

| Comando | Propósito |
| --- | --- |
| `create` | Genera un proyecto nuevo desde una configuración de stack. Es el comando por defecto. |
| `add` | Agrega addons soportados o settings de deployment a un proyecto Better Fullstack existente. |
| `history` | Muestra, exporta o limpia el historial local de scaffolds y comandos reproducibles. |
| `mcp` | Inicia el servidor MCP de Better Fullstack sobre stdio para agentes de código IA. |

## Comandos utilitarios

| Comando | Propósito |
| --- | --- |
| `docs` | Abre el sitio público de documentación de Better Fullstack en tu navegador. |
| `builder` | Abre el Stack Builder interactivo. |
| `sponsors` | Muestra sponsors de Better Fullstack. |

## Comandos de maintainers

`update-deps` es para maintainers que revisan y actualizan versiones de dependencias. Se omite intencionalmente de los workflows normales de usuario.

## Siguiente

- [Comando history](/docs/cli/history/) cubre el historial de comandos reproducibles.
