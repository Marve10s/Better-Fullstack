Better Fullstack es un generador de proyectos para stacks de producto tipados. Elige frontend, backend, capa de datos, servicios, tooling y ecosistema destino; la CLI escribe los archivos correspondientes y registra el stack en `bts.jsonc`.

Usa la CLI interactiva cuando quieras una configuración guiada, el [Stack Builder](https://better-fullstack.dev/new) cuando quieras comparar opciones visualmente, y flags explícitos cuando necesites un comando reproducible para CI, docs u otro desarrollador.

## Requisitos

La CLI corre sobre Node.js 20 o superior.

```bash
node --version
```

Los proyectos generados también pueden necesitar el toolchain de su ecosistema: tooling de Expo para React Native, Cargo para Rust, `uv` para Python, Go, Java 21, o Elixir/Erlang para proyectos Phoenix.

## Crea tu primer proyecto

Ejecuta el asistente y responde los prompts:

```npm
npm create better-fullstack@latest my-app
```

Con npm, los flags de Better Fullstack van después del separador `--`. Las tabs de package manager muestran los comandos equivalentes para pnpm, Bun y Yarn, donde ese separador no hace falta.

## Qué crean los defaults

Aceptar los defaults crea un proyecto TypeScript con:

- Frontend TanStack Router y backend Hono.
- Runtime Bun, SQLite, Drizzle, Better Auth y tRPC.
- Tailwind CSS, shadcn/ui, Vitest y Turborepo.
- El package manager usado para lanzar la CLI, versiones estables de dependencias, Git, instalacion activada y `CLAUDE.md`.

Puedes quedarte con ese stack, partir desde una plantilla o cambiar de ecosistema por completo.

## Formas de elegir un stack

| Ruta | Usala cuando |
| --- | --- |
| Asistente CLI | Quieres prompts y defaults sensatos. |
| Stack Builder | Quieres explorar combinaciones compatibles antes de generar. |
| Flags explícitos | Quieres output repetible en scripts, docs o reportes de issues. |
| Agent Skill | Quieres que un agente de código use el flujo CLI sin escribir starters a mano. |
| Servidor MCP | Quieres que un agente de IA inspeccione el schema, valide compatibilidad y haga dry-runs de forma segura. |

## Contexto del proyecto generado

Cada proyecto generado incluye:

- `bts.jsonc` con el stack seleccionado.
- Un comando reproducible impreso por la CLI.
- Archivos de framework, paquetes y configuración para el ecosistema elegido.
- Archivos opcionales de instrucciones para IA como `CLAUDE.md`, `Agents.md` o `.cursorrules`.
