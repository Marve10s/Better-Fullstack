title: 后端和 API
description: 在支持的生态系统中选择服务端框架、runtime 和 API 层。
---

后端选择决定应用逻辑在哪里运行，以及客户端如何与它通信。

## 后端框架

TypeScript 后端选项包括 Hono、Express、Fastify、Elysia、NestJS、AdonisJS、Nitro、Encore、Convex、feTS、`self` 和 `none`。

语言生态系统会暴露自己的后端界面：

| 生态系统 | 后端选择 |
| --- | --- |
| Rust | Axum、Actix Web、Rocket 或 none。 |
| Python | FastAPI、Django、Flask、Litestar 或 none。 |
| Go | Gin、Echo、Fiber、Chi 或 none。 |
| Java | Spring Boot、Quarkus 或 none。 |
| Elixir | Phoenix、Phoenix LiveView 或 none。 |

## Runtime

TypeScript 服务器可以面向 Bun、Node.js、Workers，或不使用单独 runtime。非 TypeScript 生态系统使用它们的原生工具链：Cargo、uv、Go modules、Maven/Gradle 或 Mix。

## API 层

API 选择包括 TypeScript RPC/GraphQL 选项、Rust Tonic 或 async-graphql、Python Django API/GraphQL 选项、Go gRPC，以及 Elixir REST 或 Absinthe。

## Multi-ecosystem ownership

在 multi 模式中，API、ORM 和 auth 能力会附加到被选中的后端。例如，`backend.api:rust:tonic` 属于 Rust 后端，而 `backend.orm:go:gorm` 属于 Go 后端。
