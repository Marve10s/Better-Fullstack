Auth 和支付会影响生成的 schemas、环境变量、callbacks、middleware 和 provider setup notes。

## Auth

TypeScript auth providers 包括 Better Auth、Clerk、NextAuth、Stack Auth、Supabase Auth、Auth0 和 none。

其他生态系统会在模板存在时暴露原生 helpers：

| 生态系统 | Auth 选择 |
| --- | --- |
| Rust | OAuth2 或 none。 |
| Python | Authlib、JWT 或 none。 |
| Go | Casbin、JWT helpers 或 none。GoBetterAuth 使用全局 `--auth go-better-auth` 值。 |
| Java | Spring Security 或 none。 |
| Elixir | phx.gen.auth、Ueberauth、Guardian 或 none。一些 Elixir auth 选择会在模板补齐前保持 graph-disabled。 |

## 支付

今天的支付提供商以 TypeScript 为先：Polar、Stripe、Lemon Squeezy、Paddle、Dodo 或 none。

一些 providers 需要特定的 auth 或 frontend 选择。例如，Polar 需要 Better Auth 和 Web frontend。
