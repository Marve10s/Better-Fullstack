Las opciones de UI y experiencia dan forma al código frontend generado, los primitivos de diseno y los patrones de aplicación del lado cliente.

## Styling y librerias UI

Las opciones CSS incluyen Tailwind, SCSS, Less, solo PostCSS o none. Las librerias UI incluyen shadcn/ui, shadcn-svelte, daisyUI, Radix UI, Headless UI, Park UI, Chakra UI, NextUI, Mantine, MUI, Ant Design, Base UI, Ark UI, React Aria y none.

shadcn/ui tiene presets adicionales para primitivos base, estilo visual, iconos, tema de color, color base neutral, fuente y radio.

## Formularios, validación y estado

Las opciones de aplicación frontend incluyen:

- Estado: Zustand, Jotai, Nanostores, Redux Toolkit, MobX, XState, Valtio, TanStack Store, Legend State o none.
- Formularios: TanStack Form, React Hook Form, Formik, Final Form, Conform, Modular Forms o none.
- Validacion: Zod, Valibot, ArkType, TypeBox, Typia, Runtypes, Effect Schema o none.

## Funciones de experiencia

Animacion, i18n y addons app-platform cercanos al frontend viven aqui conceptualmente, incluso cuando sus flags exactos aparecen en otras tablas de opciones.

React Native tiene sus propias opciones mobile UI: Tamagui, Gluestack UI, Uniwind, Unistyles o none.
