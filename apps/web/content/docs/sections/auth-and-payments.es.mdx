Auth y pagos afectan schemas generados, variables de entorno, callbacks, middleware y notas de setup de providers.

## Auth

Los proveedores de auth TypeScript incluyen Better Auth, Clerk, NextAuth, Stack Auth, Supabase Auth, Auth0 y none.

Otros ecosistemas exponen helpers nativos donde existen plantillas:

| Ecosistema | Opciones de auth |
| --- | --- |
| Rust | OAuth2 o none. |
| Python | Authlib, JWT o none. |
| Go | Casbin, helpers JWT o none. GoBetterAuth usa el valor global `--auth go-better-auth`. |
| Java | Spring Security o none. |
| Elixir | phx.gen.auth, Ueberauth, Guardian o none. Algunas opciones de auth de Elixir siguen deshabilitadas en el grafo hasta que las plantillas se actualicen. |

## Pagos

Los proveedores de pago hoy son TypeScript-first: Polar, Stripe, Lemon Squeezy, Paddle, Dodo o none.

Algunos providers requieren elecciones específicas de auth o frontend. Por ejemplo, Polar requiere Better Auth y un frontend web.
