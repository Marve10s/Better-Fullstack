Las opciones de calidad y tooling deciden como se verifica, documenta y mantiene el proyecto generado.

## Testing

Las opciones de testing TypeScript incluyen Vitest, Playwright, Vitest más Playwright, Jest, Cypress o none.

Otros ecosistemas exponen sus rutas nativas de testing, como librerias de testing Rust, herramientas de calidad Python, librerias de testing Java, opciones Elixir ExUnit/Mox/Bypass/Wallaby y testing mobile con Maestro o React Native Testing Library.

## Logging y observabilidad

Logging y observabilidad pueden ser específicos del stack. TypeScript tiene opciones Pino, Winston, Evlog, OpenTelemetry, Sentry y Grafana. Go y Rust tienen categorías dedicadas de logging. Java y Elixir exponen librerias relacionadas con observabilidad mediante sus opciones de ecosistema.

## Calidad de código y docs

Las opciones de calidad de código incluyen Biome, Oxlint, Ultracite, Lefthook, Husky y Ruler. Los addons de documentación incluyen Starlight y Fumadocs donde son compatibles.

## Workflow del proyecto

Package manager, version channel, docs de IA, inicializacion Git, comportamiento de install, `--dry-run` y `--verify` pertenecen a la capa de workflow en lugar del runtime de la app generada.
