Las opciones de datos deciden donde las apps generadas persisten registros, cachean trabajo, buscan contenido y almacenan archivos.

## Base de datos y ORM

La categoría compartida de base de datos incluye SQLite, PostgreSQL, MySQL, MongoDB, EdgeDB, Redis y none. En la práctica, los ORMs son específicos por ecosistema:

| Ecosistema | ORM o capa de datos |
| --- | --- |
| TypeScript | Drizzle, Prisma, Mongoose, TypeORM, Kysely, MikroORM, Sequelize o none. |
| Rust | SeaORM, SQLx, Diesel o none. |
| Python | SQLAlchemy, SQLModel, Tortoise ORM o none. |
| Go | GORM, sqlc, Ent o none. |
| Java | Spring Data JPA o none. |
| Elixir | Ecto, Ecto SQL o none. |

## Setup de provider

`--db-setup` puede agregar setup de hosting/provider para Turso, Neon, Prisma Postgres, PlanetScale, MongoDB Atlas, Supabase, Upstash, Cloudflare D1, Docker o none.

## Servicios estilo storage

Las secciones cercanas a datos incluyen:

- Cache: Upstash Redis en TypeScript más opciones de cache específicas por ecosistema donde se soportan.
- Busqueda: Meilisearch, Typesense, Elasticsearch, Algolia y rutas de busqueda específicas por ecosistema.
- File upload: UploadThing, FilePond, Uppy o none.
- File storage: S3, R2, Cloudinary o none.
