Better Fullstack 有很多选择，因此文档会按它们影响的生成产品部分来组织。这些页面解释每个分区的含义、何时选择某个选项，以及在哪里找到精确的 CLI 值。

完整选项表请使用[参考选项](/docs/reference/options/)。面向特定语言的界面请使用[生态系统](/docs/ecosystems/)。

| 分区 | 覆盖内容 |
| --- | --- |
| [前端和应用](/docs/sections/frontend-and-apps/) | Web 前端、React Native 应用、应用平台，以及前端专用约束。 |
| [后端和 API](/docs/sections/backend-and-api/) | 服务端框架、runtime、API 层，以及 multi-ecosystem 后端选择。 |
| [数据和存储](/docs/sections/data-and-storage/) | 数据库、ORM、托管设置、缓存、搜索、文件上传和文件存储。 |
| [Auth 和支付](/docs/sections/auth-and-payments/) | 身份提供商、auth 辅助工具、OAuth/JWT 路径和支付提供商。 |
| [UI 和体验](/docs/sections/ui-and-experience/) | CSS、UI 库、shadcn/ui presets、表单、验证、状态、动画和 i18n。 |
| [AI 和服务](/docs/sections/ai-and-services/) | AI SDK、email、realtime、job queues、CMS、analytics 和 feature flags。 |
| [质量和工具链](/docs/sections/quality-and-tooling/) | 测试、日志、可观测性、代码质量、docs addons、package managers、AI docs、Git 和安装行为。 |
| [部署](/docs/sections/deployment/) | Web/server 部署选择作为一个 stack 分区，而不是顶层文档区域。 |

## Solo 和 multi 模式

在 solo 模式中，一个生态系统驱动可见分类。在 [Multi-Ecosystem 模式](/docs/ecosystems/multi-ecosystem/)中，相同概念会变成 stack parts，例如 `frontend:typescript:next`、`backend:go:gin`、`backend.orm:go:gorm` 和 `database:universal:postgres`。
