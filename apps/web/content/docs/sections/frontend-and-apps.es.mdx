Esta sección controla las superficies de app de cara al usuario que genera Better Fullstack.

## Frontend web

Los frontends web TypeScript se seleccionan con `--frontend`. La categoría soporta React, Vinext, Vue/Nuxt, SvelteKit, Solid, Astro, Qwik, Angular, Redwood, Fresh y `none`.

Usa `none` para servicios solo backend. Usa modos frontend fullstack con `--backend self` cuando el framework es dueno de las rutas de servidor.

## Apps React Native

React Native es su propio ecosistema. Seleccionalo con `--ecosystem react-native` o como una parte multi-ecosystem con `mobile:react-native:native-bare`.

Las opciones mobile incluyen navegacion, UI, storage, testing, push notifications, actualizaciones OTA y deep linking. No implican categorías de backend web TypeScript ni deploy.

## Frontends Rust WASM

Rust puede generar superficies frontend Leptos o Dioxus mediante `--rust-frontend`. Esas son opciones del ecosistema Rust, separadas de los frontends web TypeScript.

## Plataformas de app

Addons de app-platform como Tauri, PWA, WXT y OpenTUI agregan capacidades de shell, offline, extension o app de terminal alrededor de stacks compatibles.
