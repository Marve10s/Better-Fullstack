这个分区控制 Better Fullstack 生成的面向用户的应用界面。

## Web 前端

TypeScript Web 前端通过 `--frontend` 选择。该分类支持 React、Vinext、Vue/Nuxt、SvelteKit、Solid、Astro、Qwik、Angular、Redwood、Fresh 和 `none`。

后端专用服务使用 `none`。当框架拥有服务端路由时，将 fullstack 前端模式与 `--backend self` 一起使用。

## React Native 应用

React Native 是独立生态系统。可以用 `--ecosystem react-native` 选择它，或在 multi-ecosystem 中作为 `mobile:react-native:native-bare` part 选择。

移动端选择包括导航、UI、storage、testing、push notifications、OTA updates 和 deep linking。它们不会隐含 TypeScript Web 后端或部署分类。

## Rust WASM 前端

Rust 可以通过 `--rust-frontend` 生成 Leptos 或 Dioxus 前端界面。这些是 Rust 生态系统选择，与 TypeScript Web 前端分开。

## 应用平台

Tauri、PWA、WXT 和 OpenTUI 等 app-platform addons 会围绕兼容 stack 添加 shell、offline、extension 或 terminal app 能力。
