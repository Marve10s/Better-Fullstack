数据选择决定生成的应用在哪里持久化记录、缓存工作、搜索内容并存储文件。

## 数据库和 ORM

共享数据库分类包括 SQLite、PostgreSQL、MySQL、MongoDB、EdgeDB、Redis 和 none。实践中 ORM 是生态系统专用的：

| 生态系统 | ORM 或数据层 |
| --- | --- |
| TypeScript | Drizzle、Prisma、Mongoose、TypeORM、Kysely、MikroORM、Sequelize 或 none。 |
| Rust | SeaORM、SQLx、Diesel 或 none。 |
| Python | SQLAlchemy、SQLModel、Tortoise ORM 或 none。 |
| Go | GORM、sqlc、Ent 或 none。 |
| Java | Spring Data JPA 或 none。 |
| Elixir | Ecto、Ecto SQL 或 none。 |

## Provider setup

`--db-setup` 可以为 Turso、Neon、Prisma Postgres、PlanetScale、MongoDB Atlas、Supabase、Upstash、Cloudflare D1、Docker 或 none 添加托管/provider setup。

## Storage 风格服务

数据相邻分区包括：

- 缓存：TypeScript 中的 Upstash Redis，以及支持时的生态系统专用缓存选择。
- 搜索：Meilisearch、Typesense、Elasticsearch、Algolia，以及生态系统专用搜索路径。
- 文件上传：UploadThing、FilePond、Uppy 或 none。
- 文件存储：S3、R2、Cloudinary 或 none。
