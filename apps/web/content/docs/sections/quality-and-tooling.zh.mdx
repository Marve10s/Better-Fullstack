质量和工具链选择决定生成项目如何被检查、记录和维护。

## 测试

TypeScript 测试选择包括 Vitest、Playwright、Vitest plus Playwright、Jest、Cypress 或 none。

其他生态系统会暴露它们的原生测试路径，例如 Rust testing libraries、Python quality tools、Java testing libraries、Elixir ExUnit/Mox/Bypass/Wallaby 选择，以及使用 Maestro 或 React Native Testing Library 的移动端测试。

## 日志和可观测性

日志和可观测性可以是 stack-specific 的。TypeScript 有 Pino、Winston、Evlog、OpenTelemetry、Sentry 和 Grafana 选择。Go 和 Rust 有专门的日志分类。Java 和 Elixir 通过它们的生态系统选项暴露可观测性相关库。

## 代码质量和文档

代码质量选择包括 Biome、Oxlint、Ultracite、Lefthook、Husky 和 Ruler。文档 addons 在兼容时包括 Starlight 和 Fumadocs。

## 项目 workflow

Package manager、version channel、AI docs、Git 初始化、install 行为、`--dry-run` 和 `--verify` 都属于 workflow 层，而不是生成应用的 runtime。
