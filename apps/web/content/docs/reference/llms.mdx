Better Fullstack publishes an `llms.txt` file so AI agents, search crawlers, and language model tools can get a compact, text-first overview of the project without parsing HTML.

- Site-level: [`/llms.txt`](https://better-fullstack.dev/llms.txt)
- Docs-specific: [`/docs/llms.txt`](https://better-fullstack.dev/docs/llms.txt)

## What it contains

The docs `llms.txt` is generated from the same metadata that builds the option reference pages. It includes:

- What Better Fullstack is and what it generates.
- Supported ecosystems (TypeScript, Rust, Python, Go, Java).
- Key CLI flags and non-interactive usage patterns.
- MCP server instructions and tool names.
- Links to option reference pages organized by category.
- Compatibility rules (tRPC/Angular/Qwik constraints, etc.).

## For agents without MCP access

If your agent can't use the MCP server, it can fetch the `llms.txt` at the start of a session for a compact context overview, then follow the links to the full reference pages for specific option values.

```bash
# Fetch the docs context
curl https://better-fullstack.dev/docs/llms.txt
```

The file is plain text, under 10 KB, and includes the most important constraints and flag names an agent needs to scaffold without interactive prompts.

## For agents with MCP access

Prefer the MCP server — the `docs://getting-started`, `docs://stack-options`, and `docs://compatibility-rules` resources give structured, queryable data rather than a static text file. See [MCP setup](/docs/ai/mcp/).
