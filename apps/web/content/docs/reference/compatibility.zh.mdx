Better Fullstack 会在生成前验证 stack choices。builder、CLI 和 MCP server 使用相同的兼容性逻辑，因此无效组合会被一致地拒绝或调整。

<CompatibilityMatrix />

## 检查内容

- 某个选项是否与选中的 baseline stack 兼容。
- 当组合被阻止时，共享的 disabled reason。
- 来自 builder 所用同一兼容性分析器的任何自动调整说明。

## 如何使用

1. 选择一个生态系统。
2. 调整与你的问题相关的 baseline stack 字段。
3. 选择要检查的分类。
4. 查看哪些选项兼容，以及被阻止的原因。

这是一个实用矩阵，并不是所有可能的 pairwise 组合。完整项目验证请运行 CLI、使用 Stack Builder，或在 scaffold 前通过 MCP 调用 `bfs_check_compatibility`。

## 兼容性行为

- 支持时，`none` 会禁用可选分类。
- 一些选择会触发自动调整，例如 Convex 替换 backend-owned database/API choices。
- 一些前端选择会有意收窄 API、UI、runtime 或 deploy 支持。
- Java、Rust、Python 和 Go 使用生态系统专用分类，而不是完整的 TypeScript stack surface。
